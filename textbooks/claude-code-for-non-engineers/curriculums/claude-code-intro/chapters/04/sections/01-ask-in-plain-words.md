## このセクションで学ぶこと

- Claude Code には専門用語ではなく「ふだんの言葉」で頼んでよいこと
- お願いするときは「何を・どうしてほしいか」をそのまま書けばよいこと
- 一度で完璧を狙わず、まずは気軽に話しかけてみればよいこと

## ふだんの言葉で話しかければよい

Claude Code を使うとき、いちばん最初につまずきやすいのが「どんなふうに頼めばいいのだろう」という不安です。プログラムを書く道具だと聞くと、なにか特別な呪文や決まった書き方が必要に思えるかもしれません。けれども、実際はその逆です。Claude Code には、同僚や友人に話しかけるのと同じ、ふだんの言葉でお願いするのがいちばんです。

たとえば「来週の会議の議事録のたたき台を作って」「このメモを読みやすく整えて」「この一覧から重複している行を消して」。こうした、職場で人にお願いするのと同じ言い回しで、そのまま伝えてかまいません。難しい言葉や英語に言い換える必要はありませんし、丁寧な敬語でなくても通じます。大切なのは「何を・どうしてほしいか」がはっきりしていることだけです。

## まずは伝えてみる、という具体例

具体的に見てみましょう。手元に「お客様アンケート.txt」というファイルがあって、その内容をざっくり知りたいとします。このとき頼む言葉は、たとえば次のようなもので十分です。

「お客様アンケート.txt を読んで、どんな意見が多かったか 3 つにまとめて。」

この一文には、Claude Code が動くために必要な情報がそろっています。「どのファイルを」「読んで」「どうまとめるか(意見を 3 つに)」が含まれているからです。逆に「いい感じにして」「よろしく」だけだと、何をすればよいか伝わらず、Claude Code も迷ってしまいます。人にお願いするときと同じで、お願いの中身が具体的なほど、返ってくる結果も期待に近づきます。

## 完璧を狙わず、気軽に始めてよい

ここで覚えておきたい注意点があります。それは、最初の一回で完璧な頼み方をしようと気負わなくてよい、ということです。Claude Code とのやりとりは一回きりの命令ではなく、何度か言葉を交わしながら仕上げていくものです(これは次のセクションから詳しく見ていきます)。ですから、最初は思いついたままに頼んでみて、返ってきた結果を見てから「もう少しこうして」と追加すればよいのです。

うまく伝わらなかったとしても、機械を壊してしまうようなことはありません。気軽に話しかけて、反応を見ながら言葉を足していく——この感覚をつかむことが、Claude Code と仲良くなる最初の一歩です。

## まとめ

- Claude Code には専門用語ではなく、ふだんの言葉でそのまま頼んでよい。
- 「何を・どうしてほしいか」を具体的に書くほど、よい結果が返ってくる。
- 最初から完璧を狙わず、まず気軽に話しかけて反応を見ればよい。
