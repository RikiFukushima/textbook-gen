## このセクションで学ぶこと

- 「最後は人が確認する」という付き合い方の大原則を理解します
- Claude Code を「相棒・アシスタント」として位置づける考え方を身につけます
- 小さく頼んで確かめる、を繰り返す進め方のコツを知ります

## 大原則は「最後は人が確認する」

得意なことと苦手なことを見てきました。この両方をふまえると、付き合い方の答えはひとつにまとまります。それは「**最後は人が確認する**」という原則です。Claude Code は作業を肩代わりしてくれますが、その結果に責任を持つのは、あくまで使う人自身です。

この原則さえ守れば、苦手分野で多少間違えても、人の最終確認で気づいて直せます。逆にこの確認を省くと、得意分野の作業でも思わぬミスを見逃してしまいます。つまり「人が確認する」というひと手間が、得意・苦手のどちらの場面でも安全装置として働いてくれるのです。

ここで言う「確認」は、難しいチェックではありません。出てきた文章を一度読んで、「言いたいことと合っているか」「おかしな点はないか」を見るだけでも十分に効果があります。完璧に検証しようと身構える必要はなく、ふだん同僚の仕事を見るときと同じ感覚で目を通せばよいのです。

## 「相棒」として位置づける

Claude Code は、命令すればなんでも完璧にこなす機械ではなく、**仕事を手伝ってくれる相棒**だと考えるとうまくいきます。新しく入ったアシスタントに仕事を頼む場面を思い浮かべてください。優秀でも、最初から丸投げはしませんよね。やってもらった結果を見て、よければ採用し、違えば直してもらう。Claude Code との付き合い方も、これとまったく同じです。

具体的なコツは「**小さく頼む**」ことです。一度に大きな仕事を任せず、短い単位に区切ってお願いし、結果を確かめながら次に進みます。こうすると、間違いがあっても早めに気づけて、手戻りが小さくて済みます。たとえば「資料を一式まとめて」と一度に丸投げするのではなく、「まず構成案を出して」「次にこの章だけ書いて」と段階を踏むイメージです。途中で方向がずれても、その場で軌道修正できるので安心です。

この進め方は、相手が AI であっても人であっても変わりません。新人に少しずつ仕事を任せて育てていくのと同じように、Claude Code とも対話を重ねながら、自分の頼み方を磨いていくのがうまく付き合うコツです。

## 注意点

「便利だから」と確認を飛ばしたくなる場面こそ要注意です。急いでいるときや、結果がそれらしく見えるときほど、つい鵜呑みにしがちです。大事な文書や、人に渡すものほど、最終確認の時間を必ず確保してください。確認の手間も含めて、トータルで時短になっているかを意識するとよいでしょう。

## まとめ

- 付き合い方の大原則は「最後は人が確認する」ことです
- Claude Code は丸投げ先ではなく、結果を見て直す「相棒」です
- 小さく頼んで都度確かめる進め方が、ミスを減らすコツです
