## このセクションで学ぶこと

- Claude Code を「モデル + 足場」として分解して見る視点を持つ
- 普段の操作の裏で 5 要素が同時に働いていることを掴む
- 観察対象として身近なエージェントを使う準備をする

## 「賢い AI」ではなく「よくできた harness」として見る

ここまでの章で、モデル単体は状態を持たない確率的関数にすぎず(第 1 章)、それを動かすには 5 つの構成要素 — ツール・コンテキスト・権限・ループ・観測 — からなる足場が要る(第 2 章)、そしてその足場の中でループが回り続けることが「自律」の正体だ(第 3 章)と見てきました。

本章では、その枠組みを **具体的なひとつのエージェントに当てはめて** 確かめます。題材は Claude Code です。普段これを使うとき、私たちはつい「賢い AI がコードを書いてくれる」と受け取りがちです。しかし harness の目で見ると、見え方が変わります。コードを書いているのはモデルですが、ファイルを実際に読み書きし、会話の流れを保ち、危険な操作の前で確認を求め、複数の作業を続けて回すのは、モデルを取り囲む **足場の側** です。

つまり Claude Code は「賢いモデル」であると同時に、第 2 章で解剖した 5 要素を実装したひとつの harness なのです。

## なぜ Claude Code が観察に向くのか

抽象的な図で 5 要素を覚えても、実際の挙動と結びつかなければ「なんとなく」のままです。Claude Code が教材として優れているのは、**5 要素のほとんどが目に見える挙動として表に出てくる** からです。

- ツール: ファイルを読み書きし、コマンドを実行する様子が画面に出る
- コンテキスト: 過去のやりとりを踏まえて応答が変わる
- 権限: 危険な操作の前に許可を求めてくる
- ループ: 一度の指示で複数の作業を続けて進める
- 観測: 何をしようとしているかが逐次表示される

しかも、あなたが今この教材を読んでいる環境そのものが Claude Code で動いているなら、観察対象は遠いどこかではなく **手元** にあります。次のセクションから、この 5 要素を一つずつ実際の挙動に結びつけて観察していきます。

## 注意点 — 「外から見える範囲」にとどめる

ひとつ釘を刺しておきます。本章で観察するのは、あくまで **利用者の側から見える挙動** です。Claude Code が内部でプロンプトをどう組み立て、どんなアルゴリズムでツールを選んでいるかといった実装の詳細は、外からは確定できません。「許可を求めてきた」「ファイルを読んだ」という観察できる事実から、harness の設計を **推し量る** のがこの章のやり方です。内部実装を断定するのではなく、挙動から構造を読む。これは既存のどんなエージェントを評価するときにも使える姿勢です。

## まとめ

- Claude Code は「賢いモデル」であると同時に、5 要素を備えたひとつの harness。
- 5 要素の多くが目に見える挙動として表に出るため、観察の教材に向く。
- 観察は外から見える挙動にとどめ、内部実装は断定せず推し量る。
