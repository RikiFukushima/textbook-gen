## このセクションで学ぶこと

- モデル単体には「記憶」「行動」「継続」の 3 つが欠けていることを理解する
- それぞれの欠如が一問一答から先に進めない原因になっていると説明できる
- これらは賢さで解決する問題ではなく、外側で補う問題だと捉える

## モデルの性質が生む 3 つの欠如

前のセクションで、モデルは確率的で状態を持たない関数であり、返せるのはテキストだけだと確認しました。この性質を実務の目線で言い換えると、モデル単体には次の 3 つが欠けています。**記憶・行動・継続**です。この 3 つが欠けているため、モデルは「一問一答」から抜け出せません。順に見ていきましょう。

## 記憶がない:毎回まっさらから始まる

まず記憶(memory)がありません。さっき自己紹介した名前も、5 分前に決めた方針も、次の呼び出しでは消えています。チャット UI で会話が続いているように見えるのは、画面の裏側で過去の発言をまとめて毎回モデルに渡し直しているからです。渡すのをやめれば、モデルは初対面の相手として振る舞います。長い作業の途中経過を「覚えておく」ことは、モデル自身にはできません。

## 行動がない:外の世界に手を出せない

次に行動(action)がありません。モデルが返せるのはテキストだけで、ファイルを保存したり、コマンドを実行したり、Web を検索したりはできません。「このファイルを書き換えておいて」と頼んでも、モデルにできるのは「書き換えた内容のテキストを出力する」ところまでで、実際にディスクへ反映する手は持っていません。外界に触れる手段がそもそも備わっていないのです。

## 継続がない:一回の応答で止まる

最後に継続(continuation)がありません。モデルは入力に対して一度応答を返したら、そこで処理が終わります。「調べて、まとめて、ファイルに保存する」という複数ステップの作業を、自分から次の手へ進めることはできません。次の一歩を踏ませるには、誰かがもう一度モデルを呼ぶ必要があります。

## 具体例と注意点:賢さでは埋まらない

たとえば「リポジトリのバグを直して」という依頼を考えます。モデルは原因の見当をつけ、修正案のコードを文章で示すところまでは見事にこなします。しかし、ファイルを実際に開いて読むこと(行動)、前のステップで読んだ内容を覚えておくこと(記憶)、テストを走らせて結果を見てもう一度直すこと(継続)は、どれも単体ではできません。

ここで大切なのは、これらが**モデルをもっと賢くすれば解決する問題ではない**という点です。記憶・行動・継続は能力の高低ではなく、関数の外側に仕組みがあるかどうかの問題です。次のセクションでは、この欠けた 3 つを外から補う発想、すなわち harness を導入します。

## まとめ

- モデル単体には記憶・行動・継続の 3 つが欠けています。
- そのため一度の応答で完結する一問一答から抜け出せません。
- これらは賢さではなく、外側の仕組みで補うべき問題です。
