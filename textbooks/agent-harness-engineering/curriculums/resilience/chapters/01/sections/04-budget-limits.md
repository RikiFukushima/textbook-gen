## このセクションで学ぶこと

- ステップ・トークン・時間という三つの予算軸とそれぞれが守るもの
- 予算は「最後の砦」であり検知の有無に関わらず必ず引くこと
- 予算超過時に静かに失敗させず、状態を残して止める設計

## 予算には三つの軸がある

前のセクションで、無限ループの検知はすり抜けることがあるので最後の砦として上限を併用する、と述べました。その「最後の砦」が予算です。予算とは、エージェントがそのタスクに使ってよい資源の上限を harness 側であらかじめ決めておくことです。検知が賢いかどうかに関係なく、予算を使い切ったら必ず止まります。

予算は性質の違う三つの軸で考えると漏れがありません。

一つめは **ステップ上限** です。ループを回してよい最大周回数を決めます。「どれだけ複雑なタスクでも 30 周で打ち切る」のように、回数そのものを縛ります。実装が最も簡単で、まずこれだけでも入れておけば暴走の周回数に天井がかかります。

二つめは **トークン予算** です。そのタスクで消費してよいモデルの入出力トークン量に上限を引きます。ステップ数が少なくても、一周ごとに巨大な観測結果を読ませていればトークンもコストも膨らみます。トークン予算はコストを直接縛る軸で、請求事故への一番効く対策です。

三つめは **タイムアウト** です。タスク開始からの経過時間で打ち切ります。外部 API の応答が返ってこない、遅い処理に巻き込まれている、といった「回数では捕まらない停滞」に効きます。

## 三つを同時に見る

これらは「どれか一つ」ではなく、いずれかが先に上限に達したら止める、という形で同時に効かせます。タスクによってどの軸が先に当たるかが変わるためです。

```python
class Budget:
    def __init__(self, max_steps, max_tokens, max_seconds):
        self.max_steps = max_steps
        self.max_tokens = max_tokens
        self.deadline = time.time() + max_seconds
        self.steps = 0
        self.tokens = 0

    def over(self):
        if self.steps >= self.max_steps:
            return "ステップ上限"
        if self.tokens >= self.max_tokens:
            return "トークン上限"
        if time.time() >= self.deadline:
            return "タイムアウト"
        return None
```

`over()` はどの軸で超過したかを返します。ループの先頭で毎回これを呼び、`None` でなければ即座に打ち切ります。三軸を一つのオブジェクトにまとめておくと、どこで止まったかも記録でき、あとで上限値の調整がしやすくなります。

## 静かに失敗させない

注意点は、予算超過を **静かな失敗にしないこと** です。上限に達したとき、ただ `return None` してエラーも残さず終わると、呼び出し側は「成功したのか、途中で力尽きたのか」を区別できません。予算で止めるときは、どの軸で超過したか、そこまでに何をしたか(中間結果や履歴)を必ず残して返します。

```python
reason = budget.over()
if reason:
    return Result(status="aborted", reason=reason, partial=agent.history)
```

こうしておけば、上限が厳しすぎたのか、タスク自体が無理筋だったのかを後から判断できます。予算は「止めるための仕組み」であると同時に、「なぜ止まったかを伝えるための仕組み」でもあると考えてください。

## まとめ

- 予算はステップ・トークン・時間の三軸で引き、いずれかが先に当たったら打ち切る。
- 予算は検知の賢さに依存しない最後の砦なので、必ず引いておく。
- 超過時は静かに失敗させず、超過理由と途中経過を残して止める。
