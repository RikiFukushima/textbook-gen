## このセクションで学ぶこと

- ヒューリスティック評価器と LLM-as-judge の違いと得意領域を説明できる
- 参照出力の有無で使える評価器が変わることを理解する
- 評価器を選ぶ際のコスト・再現性・柔軟性のトレードオフを押さえる

## 評価器とは何をするものか

評価器(Evaluator)は、Example の入力・アプリの出力・(あれば)参照出力を受け取り、スコアやラベルを返す部品です。数値(0〜1 の正答率など)を返すものも、合否やカテゴリのラベルを返すものもあります。この出力を集計することで、データセット全体での品質が一枚の数字や分布として見えるようになります。評価器は大きく 2 系統に分かれます。

## ヒューリスティック評価器

ヒューリスティック評価器は、ルールベースで決定的に判定します。代表例は次のとおりです。

- **完全一致 / 部分一致**: 出力が参照出力と一致するか。分類や抽出タスクに有効。
- **正規表現・含有チェック**: 必須キーワードが含まれるか、禁止語が無いか。
- **構造チェック**: 出力が妥当な JSON か、必要なキーが揃っているか。

これらは高速・低コストで、同じ入力なら必ず同じ結果になる(再現性が高い)のが利点です。一方で「言い回しは違うが意味は同じ」を正しく扱えません。要約の良し悪しのような曖昧な品質には向きません。

```python
def exact_match(run, example):
    prediction = run.outputs["summary"]
    reference = example.outputs["summary"]
    score = 1 if prediction.strip() == reference.strip() else 0
    return {"key": "exact_match", "score": score}
```

## LLM-as-judge

LLM-as-judge は、採点用のプロンプトを与えた LLM に出力の品質を判定させる手法です。「この要約は元の問い合わせの要点を正しく捉えているか、1〜5 で採点してください」のように指示します。意味的な正しさ・トーン・有用性といった、ルールでは書ききれない柔らかい基準を評価できるのが最大の強みです。参照出力がないタスクでも、入力と出力だけ見て採点させられます。

LangSmith には汎用の LLM-as-judge 評価器を組み立てるヘルパ(`langsmith.evaluation` の `LangChainStringEvaluator` など)が用意されており、基準(criteria)を指定するだけで評価器を作れます。次のセクションで使えるよう、ここでは `llm_judge` という名前で 1 つ用意しておきます。

```python
from langsmith.evaluation import LangChainStringEvaluator

# 「要約が問い合わせの要点を捉えているか」を判定する LLM-as-judge
llm_judge = LangChainStringEvaluator(
    "criteria",
    config={"criteria": {"faithful": "要約は元の問い合わせの要点を正しく捉えているか"}},
)
```

ただし注意点もあります。判定にもコストとレイテンシがかかり、LLM 特有のばらつきで同じ入出力でもスコアが揺れることがあります。採点基準を明確に書く、評価モデルを固定する、temperature を下げる、採点理由も出力させて検証するといった工夫で安定させます。

## 選び方のトレードオフ

実務では両者を組み合わせます。判定基準が明確で機械的に書けるもの(JSON 妥当性・必須項目の有無)はヒューリスティックで安く確実に、意味やトーンの良し悪しは LLM-as-judge で、と役割分担するのが定石です。下の表が目安です。

| 観点 | ヒューリスティック | LLM-as-judge |
| --- | --- | --- |
| コスト・速度 | 安い・速い | 高い・遅い |
| 再現性 | 高い | ばらつきあり |
| 柔軟性(曖昧な基準) | 低い | 高い |
| 参照出力 | あると有効 | 無くても可 |

## まとめ

- 評価器は出力を採点する部品で、決定的なヒューリスティックと柔軟な LLM-as-judge に大別されます。
- 機械的に判定できる基準はヒューリスティック、意味やトーンは LLM-as-judge が向きます。
- LLM-as-judge はコストとばらつきに注意し、基準の明確化とモデル固定で安定させます。
