- ストレージクラスと公開設定の注意点をつかむ

## S3 とは — ファイルを置くための専用サービス

**S3(Simple Storage Service)** は、AWS の代表的な**オブジェクトストレージ**です。前のセクションで触れたとおり、ディスクとして使う EBS とは違い、ファイルを丸ごと 1 つの単位として保存・取得します。Web サイトの画像、アプリのアップロードファイル、ログ、バックアップなど、あらゆる「ファイル置き場」に使われます。

S3 の構造はとてもシンプルで、**バケット** と **オブジェクト** の 2 つだけ覚えれば十分です。

- **バケット**: オブジェクトを入れる「入れ物」です。バケット名は**全世界で一意**である必要があり、一度作ると同じ名前を他の誰も使えません。
- **オブジェクト**: 保存する個々のデータです。データ本体・**キー(名前)**・メタデータで構成されます。

```bash
# ファイルを S3 バケットにアップロードする例
aws s3 cp ./report.pdf s3://my-company-reports/2026/report.pdf
```

キーは `2026/report.pdf` のように見えますが、S3 にフォルダの実体はなく、これは 1 本の長い名前です。スラッシュは表示上フォルダのように扱われるだけ、という点が地味につまずきやすいところです。

## 容量・耐久性という強み

S3 が広く使われる理由は、その堅牢さと手軽さにあります。

- **実質無制限の容量**: 事前に容量を確保する必要がなく、置いた分だけ自動的に増えます。
- **非常に高い耐久性**: データは複数の場所に自動複製され、設計上きわめて失われにくくなっています(いわゆる「イレブンナイン」の耐久性)。バックアップ先として信頼できます。
- **従量課金**: 保存しているデータ量と、取り出し(リクエストや転送)に対して課金されます。

さらに、アクセス頻度に応じて **ストレージクラス** を選べます。頻繁に使うデータは標準クラス、ほとんど読まないアーカイブは Glacier 系の安価なクラス、というように使い分けることでコストを下げられます。

## 公開設定の注意点

S3 で最も重要な注意点が**アクセス制御**です。バケットやオブジェクトは**既定では非公開**で、所有者だけがアクセスできます。これは安全側の初期設定です。

過去には、公開設定を誤って機密データをインターネットに晒してしまう事故が数多く起きました。「とりあえず全員に公開」にすると、URL を知る誰もがファイルを読めてしまいます。**本当に公開が必要なファイルだけを、必要な範囲で公開する**のが原則です。アクセス権限の細かな仕組み(IAM など)は第 4 章で扱いますが、まずは「S3 は既定で非公開、安易に公開しない」と覚えておきましょう。

## まとめ

- S3 はバケットとオブジェクトで構成されるオブジェクトストレージです。
- 実質無制限の容量と非常に高い耐久性を持ち、ファイル置き場やバックアップに適します。
- 既定は非公開であり、公開設定は必要最小限にとどめるのが安全の基本です。
