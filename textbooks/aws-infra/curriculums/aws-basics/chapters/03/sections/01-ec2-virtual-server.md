## このセクションで学ぶこと

- EC2 が「必要なときに借りられる仮想サーバー」であることを理解する
- インスタンスタイプ・AMI・キーペアという起動時の主要な選択肢を押さえる
- オンデマンド・スポット・リザーブドという課金方式の違いをつかむ

## EC2 とは — サーバーを「買う」のではなく「借りる」

**EC2(Elastic Compute Cloud)** は、AWS 上で**仮想サーバーを必要なだけ起動・停止できる**サービスです。第 1 章で学んだ IaaS の代表格で、「サーバーという計算資源(コンピューティング)」をクラウドから借りる仕組みだと考えてください。

従来、Web アプリケーションを動かすにはサーバー機を購入し、データセンターに設置し、OS をインストールして…という作業が必要でした。EC2 では、管理画面で数クリック、あるいは 1 つのコマンドで、数分のうちに OS が入った仮想サーバーが立ち上がります。この起動した 1 台分を **インスタンス** と呼びます。

```bash
# AWS CLI で 1 台のインスタンスを起動する例
aws ec2 run-instances \
  --image-id ami-0abcd1234efgh5678 \
  --instance-type t3.micro \
  --key-name my-key
```

## 起動時の主要な選択肢

インスタンスを起動するときは、最低限つぎの 3 つを指定します。

- **AMI(Amazon Machine Image)**: OS やソフトウェアを含む「ひな形」です。Amazon Linux や Ubuntu、Windows Server などを選べます。上記の例の `--image-id` がこれにあたります。
- **インスタンスタイプ**: CPU・メモリ・ネットワーク性能の組み合わせです。例の `t3.micro` は小規模・低コスト向け、`m5.large` は汎用、`c5.xlarge` は CPU 重視、というように用途に応じて「サイズと性格」を選びます。
- **キーペア**: インスタンスに SSH などで安全にログインするための鍵です。

これらを後から変えることもできます。たとえば負荷が増えてきたら、いったん停止して**より大きいインスタンスタイプに変更**(スケールアップ)できるのは、物理サーバーにはないクラウドならではの柔軟さです。

## 課金方式の違いに注意

EC2 は起動している時間に対して課金されます。代表的な購入オプションは次のとおりです。

- **オンデマンド**: 秒単位・時間単位で使った分だけ支払う。検証や短期利用に向く、最も手軽な方式。
- **リザーブドインスタンス / Savings Plans**: 1 年・3 年など長期利用を約束する代わりに大幅に割引される。常時稼働の本番環境向け。
- **スポットインスタンス**: AWS の空きリソースを安価に使えるが、需給によって**強制的に停止される**ことがある。中断に強いバッチ処理向け。

注意点として、**インスタンスを「停止(stop)」してもストレージ(後述の EBS)には課金が続く**こと、そして使い終わったら削除(terminate)を忘れないことが、無駄なコストを防ぐ基本です。

## まとめ

- EC2 は仮想サーバー(インスタンス)を必要なだけ起動・停止できる IaaS サービスです。
- 起動時には AMI・インスタンスタイプ・キーペアを選び、後から変更もできます。
- 課金はオンデマンド・リザーブド・スポットがあり、停止後もストレージ課金が続く点に注意します。
