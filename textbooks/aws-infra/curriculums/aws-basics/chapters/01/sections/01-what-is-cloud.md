## クラウドコンピューティングとは

クラウドコンピューティングとは、**インターネットなどのネットワーク経由で、サーバーやストレージといった IT リソースを必要なときに必要なだけ利用する仕組み**のことです。従来は、自分でコンピューターを買ってきて社内に設置し、電源やネットワークをつないで初めて使えるようになりました。クラウドではこの「物理的な機材を所有する」という前提がなくなり、事業者が用意した巨大なコンピューター群の一部を、利用者がネットワーク越しに借りて使います。

「クラウド(雲)」という言葉は、ネットワークの向こう側にある実体を意識せずに使える様子を、雲の絵で表していたことに由来します。利用者から見ると、どこにあるどんな機材で動いているのかを気にせず、サービスとして IT リソースを受け取れるのがポイントです。

クラウドを特徴づける考え方は、おおむね次の 3 つに整理できます。

- **オンデマンド(必要なときにすぐ)**: 申請して機材が届くのを待つのではなく、操作画面から数分でサーバーを立ち上げられます。
- **従量課金(使った分だけ支払う)**: 大きな初期投資をせず、利用した時間や容量に応じて料金が発生します。
- **ネットワーク経由(どこからでも)**: インターネットにつながっていれば、場所を問わず管理・利用できます。

## 具体例: 身近なクラウドサービス

クラウドは特別な技術というより、すでに私たちの生活に溶け込んでいます。たとえば写真を自動で保存してくれるオンラインのフォトサービスや、ブラウザ上で文書を編集できるオフィスツールは、いずれも手元の端末ではなく事業者のサーバー側で動いています。利用者は機材を持たず、必要なときにアクセスして、使った分だけ(あるいは月額で)支払います。

本コースで学ぶ AWS(Amazon Web Services)は、こうしたクラウドの仕組みを、企業のシステム開発者やインフラ担当者が部品のように組み合わせて使えるようにした代表的なサービス群です。

## 注意点

「クラウド=インターネット上のどこか」という曖昧なイメージのままだと、後の章でつまずきます。クラウドの実体は、世界中に置かれた事業者のデータセンター(巨大なコンピューター施設)であり、決して「実体のない魔法」ではありません。誰が機材を管理し、利用者は何を操作するのか、という責任の分担を意識することが、クラウドを正しく理解する第一歩です。この責任の違いは次のセクションで詳しく扱います。

## まとめ

- クラウドはネットワーク経由で IT リソースを必要なだけ借りて使う仕組み。
- 特徴は「オンデマンド・従量課金・ネットワーク経由」の 3 点。
- 実体は事業者のデータセンターであり、誰が何を管理するかの理解が重要。
