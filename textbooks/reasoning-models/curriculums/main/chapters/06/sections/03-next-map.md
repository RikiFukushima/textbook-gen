## このセクションで学ぶこと

- 「prompt → context → tools → loops」という技術の流れをイメージできるようになります
- 次に学ぶと良い教材として ループエンジニアリング があることを知ります
- その教材が扱うテーマの大まかな地図を持って、この教材を締めくくります

## AIの使いこなしには系譜がある

ここまでで、あなたは「なぜAIが賢くなったのか」——推論の原理——をつかみました。最後に、その先に続く道の地図をお渡しします。

AIをうまく使う技術は、次のような順で発展してきました。

- prompt(プロンプト): 一問一答の頼み方を工夫する段階。ただし一回のやり取りには限界があります
- context(文脈): 何をどう伝えるか、渡す情報そのものを設計する段階
- tools(道具): AIに検索や計算などの道具を使わせ、できることを広げる段階
- loops(ループ): AIに計画→実行→検証を自分でくり返させ、自走させる段階

前のセクションで見たエージェントは、この一番奥の loops にあたります。プロンプトの工夫から始まった流れが、いまや「自律的に回るループをどう組むか」というところまで来ているのです。

面白いのは、後ろの段階ほど前の段階を捨てるわけではない点です。ループを組むときも、その中では良いプロンプトや適切な文脈、便利な道具がやはり必要になります。積み上げてきたものの上に、新しい層が乗っていくイメージです。だからこそ、いま学んだ推論の理解も、次の段階でそのまま生きてきます。

## 次に読むと繋がる — ループエンジニアリング

その loops を"使う側"から"組み立てる側"に回るための技術が ループエンジニアリング です。同じ教材群に「ループエンジニアリング入門」が用意されているので、この教材の次の一歩にちょうど良いでしょう。

深追いはしませんが、そこで扱うテーマだけ地図として予告しておきます。

- ループの解剖学: 自走するループがどんな部品でできているかを分解して見る
- メモリ/状態の永続化: 長く動くループが「途中経過」を覚えておくしくみ
- maker-checker: 作る側と検証する側を分け、成果物を別の目で確かめる考え方
- 停止条件の設計: ループをいつ止めるかという、暴走させないための取り決め

## この教材の締めくくり

この教材で得たのは、「なぜAIが賢くなったか」という土台の理解です。推論モデルが段階を踏んで考え、その賢さがエージェントの自走を支えている——ここまで見えていれば十分です。

次はその賢さを使って「どうシステムを組むか」の番です。地図はお渡ししました。興味が向いたら、ループエンジニアリングへ進んでみてください。

## まとめ

- AIの使いこなしは prompt → context → tools → loops と発展してきました
- 一番奥の loops を組み立てる技術が ループエンジニアリング です
- 「なぜ賢くなったか」を掴んだ次は「その賢さでどう組むか」へ、と締めくくります
