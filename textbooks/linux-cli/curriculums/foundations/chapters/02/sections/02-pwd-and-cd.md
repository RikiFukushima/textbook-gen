## このセクションで学ぶこと

- CLI には「いま自分がいる場所」=カレントディレクトリという概念があること
- `pwd` で現在地を確認し、`cd` で移動するという基本動作
- 引数なしの `cd`、`cd ..`、`cd -` という 3 つの便利な移動方法

## CLI にも「現在地」がある

GUI でフォルダを開いているとき、ウィンドウには「いまどのフォルダを見ているか」が表示されています。CLI にもまったく同じ概念があり、シェルは常に「いま自分がどのディレクトリにいるか」を覚えています。この現在地を**カレントディレクトリ**(作業ディレクトリ)と呼びます。

カレントディレクトリが重要なのは、多くのコマンドが「いまいる場所」を基準に動くからです。たとえば `ls` を引数なしで実行すると、カレントディレクトリの中身が表示されます。同じ `ls` でも、いる場所が違えば結果は変わります。CLI で迷子になったと感じたら、原因のほとんどは「自分がどこにいるか見失っている」ことです。

## pwd — 現在地を確認する

現在地を表示するコマンドが `pwd`(print working directory)です。引数もオプションも要りません。

```bash
pwd
```

実行すると、たとえば次のように表示されます。

```text
/home/taro
```

これは「ルート(`/`)の下の home の下の taro にいる」という意味です。この `/` 区切りの住所表記(パス)の読み方は次章でじっくり扱うので、ここでは「`pwd` を打てば現在地の住所が出る」とだけ押さえれば十分です。

## cd — 現在地を変える

移動には `cd`(change directory)を使います。引数に行き先のディレクトリを指定します。

```bash
cd /tmp
pwd
```

```text
/tmp
```

ここで初心者がよく戸惑うのが、**`cd` は成功しても何も表示しない**ことです。エラーが出なければ移動は成功しています。CLI の世界には「成功したときは黙っている(No news is good news)」という流儀があり、`cd` はその代表例です。不安なら、移動のたびに `pwd` で確認する癖をつけましょう。

`cd` には覚えておくと便利な使い方が 3 つあります。

```bash
cd ..      # 1 つ上のディレクトリへ移動する
cd         # 引数なし: 自分のホームディレクトリへ戻る
cd -       # 直前にいたディレクトリへ戻る
```

- `..` は「1 つ上の階層」を表す特別な書き方です(詳しくは次章で扱います)。
- 引数なしの `cd` は、どこにいても自分のホームディレクトリ(`/home/taro` のような自分専用の場所)へ一発で帰れる「家に帰るボタン」です。
- `cd -` は「さっきの場所に戻る」で、2 つの場所を行き来する作業で重宝します(このときだけは例外的に、移動先のパスが表示されます)。

## 注意点

- 存在しない場所を指定すると `No such file or directory`(そのようなファイルやディレクトリはありません)というエラーが出ます。たいていは**打ち間違い**が原因なので、まずスペルとスペースの位置を確認してください。
- `cd` で移動できるのは**ディレクトリだけ**です。ファイル名を指定すると `Not a directory` というエラーになります。
- 迷子になったら、慌てず `pwd` で現在地を確認し、だめなら引数なしの `cd` でホームに帰る。この「現在地確認 → 帰宅」の 2 手を覚えておけば、CLI で遭難することはありません。

## まとめ

- シェルは常に「現在地」=カレントディレクトリを持ち、多くのコマンドはそこを基準に動く
- `pwd` で現在地を表示し、`cd 行き先` で移動する。`cd` は成功すると何も表示しない
- `cd ..` で 1 つ上へ、引数なしの `cd` でホームへ、`cd -` で直前の場所へ戻れる
