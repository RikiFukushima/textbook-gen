## このセクションで学ぶこと

- `ls` でカレントディレクトリの中身を見る方法と、定番オプション `-l`・`-a`
- 知らないコマンドに出会ったときに `--help` で素早く調べる方法
- `man` でじっくりマニュアルを読む方法と、`q` での閉じ方

## ls — まわりを見回す

`pwd` が「現在地の確認」なら、`ls`(list)は「まわりの見回し」です。引数なしで実行すると、カレントディレクトリにあるファイルとディレクトリの名前が並びます。

```bash
ls
```

```text
docs  memo.txt  photos
```

前のセクションで学んだ文法どおり、オプションと引数も付けられます。実務で特によく使うのが次の 2 つのオプションです。

```bash
ls -l     # サイズ・更新日時・権限つきの詳細表示
ls -a     # 隠しファイル(. で始まるファイル)も表示
ls -la    # 両方を同時に指定
```

Linux では、名前がドット(`.`)で始まるファイルは**隠しファイル**として扱われ、普通の `ls` では表示されません。設定ファイルの多くがこの形式なので、「あるはずのファイルが見えない」ときは `-a` を思い出してください。また引数にディレクトリを指定すれば、移動せずに `ls /tmp` のように他の場所の中身ものぞけます。

## --help — その場で素早く調べる

ここからがこのセクションの本題です。CLI に強い人は、コマンドを全部暗記しているわけではありません。**調べ方を知っている**だけです。

最初の調べ方が `--help` オプションです。多くのコマンドは、`--help` を付けて実行すると、使い方の要約(文法・主なオプションの一覧)を表示してくれます。

```bash
ls --help
```

出力には `Usage: ls [OPTION]... [FILE]...` のような書式と、オプションの一覧が並びます。`[ ]` で囲まれた部分は「省略可能」という意味の慣習です。全部読む必要はなく、「目当ての機能がありそうか」を眺める使い方で十分です。

## man — じっくり読むマニュアル

もう 1 つの調べ方が `man`(manual)コマンドです。引数に調べたいコマンド名を渡します。

```bash
man ls
```

`--help` が「要約のチートシート」だとすると、`man` は「正式な取扱説明書」です。説明・全オプション・注意事項まで網羅されています。

`man` を実行すると、画面全体がマニュアル表示に切り替わります。これはページャと呼ばれる閲覧モードで、操作方法が普段と変わります。

- **矢印キーまたは Enter** で 1 行ずつ、**スペースキー**で 1 画面ずつ進む
- **`q`** を押すと終了して元の画面に戻る

この「`q` で抜ける」は次のセクションでも扱う重要な脱出キーです。まずは `man ls` を開いて、スペースで眺めて、`q` で戻る、という一連の動きを体で覚えてください。

## 注意点

- `--help` の表示はコマンドごとに書きぶりが違い、macOS など環境によっては `--help` が使えないコマンドもあります。その場合は `man` を試すのが定石です。
- `man` は英語で表示されることが多いですが、最初の Usage(書式)とオプション一覧だけ拾い読みすれば実用上は足ります。完読しようとしないでください。
- 暗記より検索です。「`ls -何だっけ?`」となったら、その場で `ls --help` を打つほうが速く、確実です。

## まとめ

- `ls` でディレクトリの中身を一覧でき、`-l` で詳細、`-a` で隠しファイルまで見える
- 素早く調べるなら `--help`、じっくり調べるなら `man`。暗記ではなく調べ方を覚える
- `man` の画面はスペースで進み、`q` で抜ける
