## このセクションで学ぶこと

- 絶対パスは `/` から、相対パスは現在地から場所を指定することを理解する
- `.`(現在地)・`..`(1 つ上)・`~`(ホームディレクトリ)の 3 つの記号を使えるようになる
- 「確実さの絶対パス、手軽さの相対パス」という使い分けの感覚をつかむ

## 住所の書き方は 2 通りある

前のセクションで、パスは「ファイルの住所」だと学びました。実はこの住所には 2 通りの書き方があります。

1 つめが絶対パスです。必ず `/` から書き始め、ルートからの道のりをすべて書きます。「東京都千代田区 1-1-1」のような正式な住所で、誰がどこで読んでも同じ場所を指します。

2 つめが相対パスです。`/` 以外から書き始め、現在地(カレントディレクトリ)を起点とした道のりを書きます。「ここから 2 軒隣」のような道案内で、短く書ける代わりに、自分が今どこにいるかで意味が変わります。

相対パスと合わせて覚えておきたい便利な記号が、次の 3 つです(`~` はシェルが自分のホームディレクトリの絶対パスに展開する記号で、厳密には相対パスではありませんが、パス指定を短くする仲間としてここで一緒に押さえます)。

| 記号 | 意味 |
| --- | --- |
| `.` | 現在地そのもの |
| `..` | 1 つ上の階層(親ディレクトリ) |
| `~` | 自分のホームディレクトリ |

ホームディレクトリとは、ユーザーごとに用意された自分専用の場所です。ユーザー taro なら `/home/taro` がホームで、ログイン直後はここからスタートします。

## 実際に動いて確かめる

`cd` と `pwd` を使って、2 通りの書き方を体感してみましょう。

```bash
$ pwd
/home/taro
$ cd notes        # 相対パス: 現在地の中の notes へ
$ pwd
/home/taro/notes
$ cd ..           # 相対パス: 1 つ上へ戻る
$ pwd
/home/taro
$ cd /var/log     # 絶対パス: どこにいても同じ場所へ
$ pwd
/var/log
$ cd ~            # ホームへ帰る(cd だけでも同じ)
$ pwd
/home/taro
```

`cd notes` は、現在地が `/home/taro` だから `/home/taro/notes` に着きました。もし `/tmp` にいる状態で同じコマンドを打てば `/tmp/notes` を探しに行き、なければエラーになります。同じ文字列でも現在地次第で結果が変わる — これが相対パスの性質です。

`..` は重ねて使えます。たとえば `cd ../..` で 2 階層上に一気に戻れます。`.` は「移動」ではあまり使いませんが、「今いる場所のこのファイル」と明示したいときに `./script.sh` のような形で頻繁に登場するので、見かけたら「現在地の」と読み替えてください。

## 注意点 — 相対パスは「いつでも動く」とは限らない

相対パスは打つ文字数が少なく日常の移動では主役ですが、現在地が前提になっている点に注意が必要です。手順書に `cd notes` とだけ書いてあっても、どこで実行するかが共有されていなければ再現できません。他人にコマンドを伝えるときや、あとで見返すメモに残すときは、`/home/taro/notes` のような絶対パスで書いておくと迷いがなくなります。

また `~` は、シェルが「そのユーザーのホーム」に展開する記号です。taro が打てば `/home/taro`、hanako が打てば `/home/hanako` を指します。便利な一方、ユーザーが変われば指す場所も変わることは頭の片隅に置いておきましょう。

## まとめ

- 絶対パスは `/` 起点でどこから打っても同じ場所、相対パスは現在地起点で短く書ける
- `.` は現在地、`..` は 1 つ上、`~` は自分のホームディレクトリ
- 日常の移動は相対パス、他人への共有や記録には絶対パスが向く
