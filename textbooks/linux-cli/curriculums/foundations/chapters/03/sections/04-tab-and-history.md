## このセクションで学ぶこと

- Tab 補完でパスやコマンド名を途中まで打って補わせる操作を身につける
- 矢印キー・`history`・Ctrl+R でコマンド履歴を呼び出せるようになる
- 補完が効かないことを「タイプミスや存在しないパスのサイン」として使えるようになる

## 速い人は「打って」いない

ここまでパスの読み書きを学んできて、「`/var/log/nginx/access.log` みたいな長い文字列を毎回手で打つのか」と気が遠くなった人もいるかもしれません。安心してください。CLI に慣れた人ほど、実はほとんど打っていません。シェルに「続きを補わせる」「過去を再利用する」のが基本動作だからです。その道具が Tab 補完とコマンド履歴です。

Tab 補完は、コマンド名やパスを途中まで入力して Tab キーを押すと、シェルが残りを補ってくれる機能です。履歴は、過去に打ったコマンドの記録を呼び出して再利用する機能です。手で全部打つのは、遅いだけでなくタイプミスの温床でもあります。「打たないが勝ち」は速さと正確さ、両方のための合言葉です。

## 具体例 — Tab と履歴を使ってみる

まず Tab 補完です。`/var/log` へ移動したいとき、次のように打ちます。

```bash
$ cd /v          # ここで Tab を押す
$ cd /var/       # ← シェルが補ってくれた
$ cd /var/lo     # さらに lo まで打って Tab
$ cd /var/log/
```

途中で Tab を押しても反応がないときは、候補が複数あって絞り込めていません。そこで Tab をもう一度(計 2 回)押すと、候補の一覧を見せてくれます。

```bash
$ cd /var/l      # Tab を 2 回押す
lib/  local/  lock/  log/
```

一覧を見て 1 文字足し、また Tab。この「ちょい打ち → Tab」のリズムが身につくと、長いパスもほぼ Tab だけで入力できます。

次に履歴です。いちばん手軽なのは上下の矢印キーで、上キーを押すたびに 1 つ前のコマンドが呼び出されます。少し前のものを探すなら `history` で一覧を表示できます。

```bash
$ history
  98  cd /var/log
  99  ls /etc/nginx
 100  pwd
```

そして強力なのが Ctrl+R です。押してからキーワードを打つと、それを含む過去のコマンドを新しい順に検索してくれます。

```bash
(reverse-i-search)`ngi': ls /etc/nginx
```

この状態で Enter を押せばそのまま実行、もう一度 Ctrl+R を押せばさらに過去の候補へ遡れます。「先週打ったあの長いコマンド」を 3 秒で呼び戻せる、実務での使用頻度がきわめて高い操作です。

## 注意点 — 補完が効かないのは「警告」でもある

Tab 補完には、入力支援以上の価値があります。補完はシェルが実在するファイル名から候補を探す仕組みなので、Tab を押しても何も補完されないなら、そのパスが存在しないか、途中にタイプミスがある可能性が高いのです。「実行してエラーで気づく」前に「補完されなくて気づく」。これは事故を未然に防ぐ習慣として意識する価値があります。

なお、補完や履歴の細かい挙動はシェル(bash や zsh)や設定によって少し異なります。ただ「Tab で補完、矢印と Ctrl+R で履歴」という基本はどの環境でも通用するので、まずこの 3 つを手に馴染ませてください。

## まとめ

- パスやコマンド名は手で打ち切らず、途中まで打って Tab で補わせる。Tab 2 回で候補一覧
- 過去のコマンドは矢印キー・`history`・Ctrl+R で再利用する
- 補完が効かないのは存在しないパスやタイプミスのサイン。事故防止にも効く
