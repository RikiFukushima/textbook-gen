## このセクションで学ぶこと

- プロンプトは、シェルが「入力を待っています」と知らせる合図であること
- プロンプトにはユーザー名・ホスト名・現在地などの情報が詰まっていること
- 行末の記号 `$` は一般ユーザー、`#` は管理者(root)を表すこと

## プロンプトとは何か

ターミナルを開くと、黒い画面に文字列が表示され、その横でカーソルが点滅します。

```
taro@my-machine:~/projects$
```

この文字列が **プロンプト** です。prompt は「促す」という意味の英語で、シェルが「コマンドをどうぞ」と入力を促している状態を表します。逆に言えば、プロンプトが表示されていない間は、前のコマンドがまだ実行中だということです。「プロンプトが返ってくる = 処理が終わり、次の入力を待っている」という感覚は、CLI 操作の基本のリズムになります。

## プロンプトの読み方

先ほどの例を分解してみましょう。表示の形は環境によって多少違いますが、典型的には「誰が・どのマシンの・どこにいるか」が並んでいます。

```
taro @ my-machine : ~/projects $
 │        │             │      │
 │        │             │      └─ 一般ユーザーの合図
 │        │             └─ 現在地(いまいるディレクトリ)
 │        └─ ホスト名(マシンの名前)
 └─ ユーザー名
```

- **ユーザー名**: いまどのユーザーとして操作しているか
- **ホスト名**: どのマシンを操作しているか。SSH で別のサーバーに入ると、ここが接続先の名前に変わります
- **現在地**: シェルがいま居るディレクトリ。`~` は自分のホームディレクトリを表す記号です(パスの読み方は第 3 章で詳しく学びます)

遠隔操作が当たり前の実務では、「手元の PC のつもりで打ったら本番サーバーだった」という事故が実際に起こります。コマンドを打つ前にプロンプトのホスト名を一瞥する癖は、その事故を防ぐ安全装置になります。

## $ と # — 行末の記号は権限の合図

プロンプト末尾の記号にも意味があります。

- `$`(シェルによっては `%`): 一般ユーザーとして操作中
- `#`: **root**(管理者・スーパーユーザー)として操作中

root はシステムのあらゆる操作が許される特別なユーザーで、システムファイルの削除すら止められません。`#` が見えているときは「ブレーキのない車に乗っている」状態だと考えて、一段慎重に操作するのが実務の作法です。普段の作業は `$` の一般ユーザーで行い、必要なときだけ管理者の力を借りるのが基本スタイルです。

## 注意点: ドキュメントの `$` は打ち込まない

技術記事やマニュアルでは、コマンド例の行頭に `$` が書かれていることがよくあります。

```
$ ls -l
```

この `$` は「ここからがコマンドです」と示すためにプロンプトを再現した飾りであって、コマンドの一部ではありません。コピーするのは `ls -l` の部分だけです。`$` ごと貼り付けるとエラーになる——これは初学者が最初に踏む定番の落とし穴なので、先に知っておきましょう。

## まとめ

- プロンプトはシェルの「入力待ち」の合図で、表示が戻るまで前の処理は実行中
- ユーザー名・ホスト名・現在地が読み取れ、特にホスト名は遠隔操作時の事故防止に役立つ
- 行末の `$` は一般ユーザー、`#` は root の合図。ドキュメント上の `$` はコピーしない
