## このセクションで学ぶこと

- シンボリックリンクが「パスを書いた小さなファイル」であることを理解する
- `ln -s` でリンクを作り、`ls -l` で見分けられるようになる
- リンク切れなど、シンボリックリンク特有の落とし穴を知る

## シンボリックリンクとは — 実体を指す「矢印」

前のセクションで、Linux ではいろいろなものがファイルの顔をして並んでいることを見ました。シンボリックリンク(symbolic link、symlink)もその仲間で、**「別のファイルやディレクトリへのパス」だけを中身に持つ特殊なファイル**です。

イメージとしては「矢印」や「ショートカット」が近いでしょう。リンク自体はデータの実体を持たず、「本物はあちらにあります」と指し示すだけです。リンクを開けば指し先のファイルが開き、リンク越しに書き込めば指し先が書き換わります。

```mermaid
graph LR
    A["current(シンボリックリンク)"] -- "指し示す" --> B["releases/v2.0/(実体のディレクトリ)"]
    C["利用者・プログラム"] -- "current を開く" --> A
```

実務でよく見るのは、**「最新版」への入り口を固定する**使い方です。たとえばアプリのバージョンを `v1.0`、`v2.0` とディレクトリで管理し、`current` というリンクを最新版に向けておきます。新版をリリースしたらリンクの向き先を差し替えるだけで、利用側は常に `current` を見ればよい、という構成にできます。

## 具体例 — 作って、見分けて、たどる

シンボリックリンクは `ln -s` で作ります。**「実体が先、リンク名が後」**という引数の順番がポイントです。

```bash
echo "hello" > original.txt
ln -s original.txt shortcut.txt   # ln -s 実体 リンク名
cat shortcut.txt
# hello
```

リンク経由で読むと、指し先である `original.txt` の中身が表示されました。リンクかどうかは `ls -l` で見分けられます。

```bash
ls -l
# -rw-r--r-- 1 user user 6 ... original.txt
# lrwxrwxrwx 1 user user 12 ... shortcut.txt -> original.txt
```

行頭が `l` で始まり、末尾に `-> original.txt` と**指し先が矢印付きで表示される**のがシンボリックリンクです。行頭の 1 文字はファイルの種別を表していて、普通のファイルは `-`、ディレクトリは `d`、そしてシンボリックリンクは `l` という 3 つ目の顔ぶれになります。

## 注意点 — リンク切れと削除のふるまい

シンボリックリンクには、初学者がつまずきやすいポイントが 2 つあります。

1 つ目は**リンク切れ**です。リンクは「パスを覚えているだけ」なので、指し先が削除・移動されてもリンク自体は残ります。

```bash
rm original.txt
cat shortcut.txt
# cat: shortcut.txt: No such file or directory
```

`shortcut.txt` は `ls` には見えているのに、開こうとするとエラーになります。指し先を失った矢印、これがリンク切れです。

2 つ目は**削除のふるまい**です。`rm shortcut.txt` のようにリンクを `rm` で消しても、消えるのはリンクだけで実体は無事です。逆に、実体を消せば(たとえリンクが残っていても)データは失われます。「リンクを消す操作」と「実体を消す操作」は別物だと覚えておきましょう。

## まとめ

- シンボリックリンクは、指し先のパスを中身に持つ「矢印」のような特殊ファイル
- `ln -s 実体 リンク名` で作成し、`ls -l` では行頭の `l` と `->` 付きの表示で見分けられる
- 指し先が消えるとリンク切れになる。リンクの削除は実体に影響しないが、実体の削除はデータを失う
