## このセクションで学ぶこと

- `find` の基本形「`find 起点 条件`」でファイルを探せるようになる
- `-name` や `-type` といった代表的な条件を組み合わせる
- 探索範囲の指定ミスや、よく出る権限エラーへの対処を知る

## find の考え方 — 起点から下をすべてたどる

ファイルが増えてくると、「あの設定ファイル、どこに置いたっけ」という場面が必ず訪れます。そこで使うのが `find` です。

`find` の動きはシンプルで、**指定した起点ディレクトリから下の階層をすべてたどり、条件に合うものを表示する**というものです。基本形はこうです。

```bash
find 起点ディレクトリ 条件
```

「どこから探すか」を最初に書き、「何を探すか」を条件で続けます。この語順さえ押さえれば、あとは条件のバリエーションを覚えていくだけです。

## 具体例 — 名前で探す、種類で絞る

もっともよく使う条件が、名前で絞る `-name` です。

```bash
find . -name "notes.txt"
# ./projects/memo/notes.txt
```

`.`(現在のディレクトリ)を起点に、`notes.txt` という名前を探しました。名前が正確に思い出せないときは、ワイルドカード `*` を使ったパターンで探せます。

```bash
find . -name "*.log"
# ./app/error.log
# ./app/access.log
```

`*.log` のようなパターンは**引用符で囲む**のがお作法です。囲まないと、`*` がシェルによって先に展開されてしまい、意図しない結果になることがあります。

種類で絞りたいときは `-type` を使います。`f` がファイル、`d` がディレクトリです。条件は並べて書くと「かつ」で組み合わされます。

```bash
find . -type d -name "config"   # config という名前のディレクトリだけ
find /var/log -type f -name "*.log"   # /var/log 以下のログファイルだけ
```

ほかにも `-mtime -1`(1 日以内に更新)や `-size +100M`(100MB 超)など、時間やサイズの条件もあります。全部を暗記する必要はなく、「find は条件を足して絞り込める」と知っていれば、必要なときに調べられます。

## 注意点 — 起点の指定と Permission denied

`find` でつまずきやすいのは次の 2 点です。

- **起点を広く取りすぎない**。`find /` のようにルートから探すと、システム全体を走査するため時間がかかります。まずは `.` や `~` など、心当たりのある範囲から始めましょう。
- **`Permission denied` は異常ではない**。他人の領域やシステム領域など、読む権限のないディレクトリに当たるとエラーが表示されます。探索自体は続いているので、結果に目的のものがあれば気にしなくて構いません。エラー表示が邪魔な場合は、この章の最初のセクションで登場した `/dev/null` の出番です(`2> /dev/null` でエラー出力だけ捨てられます。リダイレクトの詳細は後のカリキュラムで扱います)。

## まとめ

- `find` は「`find 起点 条件`」の形で、起点ディレクトリ以下を再帰的に探す
- `-name`(名前・パターン)と `-type`(ファイル/ディレクトリ)が代表的な条件で、並べると「かつ」で絞り込める
- 起点は狭めに取り、`Permission denied` は「読めない場所があった」だけなので慌てない
