## このセクションで学ぶこと

- `chmod` の記号指定(`u+x` など)で権限を追加・削除できる
- 数値指定(755・644)の計算方法を理解する
- `chown` で所有者とグループを変更できることを知る

## chmod — 記号で「誰に・何を」指定する

権限を読めるようになったら、次は変更です。権限の変更は `chmod`(change mode)で行います。最も読みやすいのは記号指定で、**「誰に(u/g/o/a)+ どうする(+/-)+ 何を(r/w/x)」**の順に並べます。`u` は所有者(user)、`g` はグループ、`o` はその他、`a` は全員です。

実務で一番よく使うのは、**書いたスクリプトに実行権限を付ける**操作です。作ったばかりのファイルには実行権限がないため、そのままでは実行できません。

```bash
ls -l deploy.sh
# -rw-r--r-- 1 alice developers 120 Jun 13 10:00 deploy.sh
./deploy.sh
# bash: ./deploy.sh: Permission denied

chmod u+x deploy.sh
ls -l deploy.sh
# -rwxr--r-- 1 alice developers 120 Jun 13 10:00 deploy.sh
./deploy.sh   # 実行できる
```

逆に削るときは `-` を使います。たとえば `chmod go-r secret.txt` とすれば、グループとその他から読み取り権限を外せます。

## 数値指定 — r=4・w=2・x=1 の足し算

もう 1 つの書き方が数値指定です。**r=4・w=2・x=1** と決まっていて、1 区分の権限はその合計 1 桁で表せます。`rwx` は 4+2+1=7、`r-x` は 4+1=5、`r--` は 4 です。これを所有者・グループ・その他の順に 3 桁並べます。

```bash
chmod 755 deploy.sh   # rwxr-xr-x(所有者は全部、他は読みと実行)
chmod 644 report.txt  # rw-r--r--(所有者は読み書き、他は読みだけ)
chmod 600 id_rsa      # rw-------(所有者以外は何もできない)
```

755・644・600 の 3 つは定番パターンです。特に 600 は SSH の秘密鍵など「自分以外に見せてはいけないファイル」の定番で、権限が緩いと SSH 側が接続を拒否することもあります。

## chown — 所有者を変える

所有者やグループそのものを変えるのは `chown`(change owner)です。`所有者:グループ` の形で指定します。

```bash
sudo chown alice:developers report.txt
```

頭に `sudo` が付いているのがポイントです。**所有者の変更は通常、管理者権限(root)がなければ実行できません**。誰でも所有者を変えられたら、他人のファイルを自分のものにできてしまうからです。この `sudo` の正体は次のセクションで扱います。

## 注意点 — 777 と -R は事故のもと

`chmod` でつまずきやすいのが、`Permission denied` を見て反射的に `chmod 777`(全員に全権限)としてしまうことです。エラーは消えますが、誰でも書き換え・実行できる危険なファイルが残ります。必要な区分に必要な権限だけ足すのが原則です。

また、`chmod -R` や `chown -R` の `-R` はディレクトリ以下を**再帰的に一括変更**します。便利な反面、対象を間違えるとシステム全体の権限を壊すこともあるため、実行前にパスをよく確認しましょう。

## まとめ

- `chmod u+x file` のように「誰に・どうする・何を」で権限を増減できる
- 数値指定は r=4・w=2・x=1 の合計を 3 桁並べる(755・644・600 が定番)
- `chown` は所有者・グループの変更で、通常 sudo が必要。777 と -R の乱用は避ける
