## このセクションで学ぶこと

- 変数の定義(`NAME=値`)と参照(`$NAME`)の書き方
- `$1`・`$2` でスクリプトの引数を受け取る方法
- `$@`(全引数)・`$#`(個数)などの特別な変数

## 変数 — スクリプトの中のメモ

スクリプトの中では、値に名前を付けて使い回せます。これが **変数** です。`NAME=値` で定義し、`$NAME` で参照します。

```bash
#!/bin/bash
NAME="Linux"
echo "Hello, $NAME"        # Hello, Linux
echo "File: ${NAME}_note"  # File: Linux_note
```

直後に別の文字をつなげるときは `${NAME}` と波かっこで名前の範囲を明示します。また、`$(コマンド)` と書くと **コマンドの実行結果を値として取り出せます**(コマンド置換)。日付入りのファイル名などで大活躍する書き方です。

```bash
TODAY=$(date +%Y%m%d)
echo "今日は $TODAY"   # 今日は 20260613
```

## 引数 — 呼び出すたびに違う値を渡す

毎回同じ動きしかしないスクリプトより、「対象だけ変えて使い回せる」スクリプトのほうが道具として便利です。実行時にスクリプト名の後ろに書いた値は **引数** としてスクリプトに渡り、1 番目が `$1`、2 番目が `$2`… という **位置パラメータ** で受け取れます。

```bash
#!/bin/bash
echo "こんにちは、$1 さん"
echo "引数は全部で $# 個: $@"
```

```bash
chmod +x greet.sh
./greet.sh Tanaka Sato
# こんにちは、Tanaka さん
# 引数は全部で 2 個: Tanaka Sato
```

位置パラメータの仲間をまとめておきます。

| 変数 | 意味 |
| --- | --- |
| `$0` | スクリプト自身の名前 |
| `$1` `$2` … | 1 番目・2 番目…の引数 |
| `$#` | 引数の個数 |
| `$@` | すべての引数 |

`$@` は「受け取った引数を全部まとめて扱いたい」ときに使います。次のセクションの `for` と組み合わせると、渡されたファイル全部に同じ処理をする、といった書き方ができます。

## 注意点

- **`=` の前後に空白を入れてはいけません**。`NAME = "Linux"` と書くと、シェルは `NAME` というコマンドの実行と解釈してエラーになります。初学者が最初に踏む地雷です。
- 変数や引数を参照するときは `"$1"` のように **ダブルクォートで囲む癖**を付けてください。空白を含むファイル名が渡されたときに、裸の `$1` は途中で 2 つの引数に割れてしまいます。
- 引数が渡されなかった場合、`$1` は空文字列になります。「引数があるか確かめてから動く」書き方は次のセクションの `if` で扱います。

## まとめ

- 変数は `NAME=値` で定義し `$NAME` で参照。`=` の前後に空白を入れない
- 引数は `$1`・`$2` で受け取り、`$#` が個数、`$@` が全引数
- 参照は `"$1"` のようにダブルクォートで囲むのが安全な作法
