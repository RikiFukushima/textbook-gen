## このセクションで学ぶこと

- コマンドの並びをファイルに保存して「再生」する、シェルスクリプトの考え方
- 1 行目に書く shebang(`#!/bin/bash`)の意味
- `chmod +x` で実行権限を付けて、`./` 付きで実行する手順

## スクリプト = コマンドを書き溜めたファイル

ここまでの章で、パイプやリダイレクトを組み合わせれば 1 行でかなりの仕事ができることを見てきました。しかし毎回同じ長いコマンドを打ち直すのは無駄です。そこで、**コマンドの並びをテキストファイルに保存しておき、あとから何度でも再生する**のがシェルスクリプトです。中身は普段ターミナルに打っているコマンドそのもので、特別な言語を新しく覚える必要はありません。

まずは最小のスクリプトを作ってみます。エディタで `hello.sh` というファイルを作り、次の 3 行を書きます。

```bash
#!/bin/bash
echo "Hello, script!"
date
```

## shebang — 1 行目の宣言

1 行目の `#!/bin/bash` を **shebang(シバン)** と呼びます。これは「このファイルは `/bin/bash` というプログラムに実行させてください」という宣言です。スクリプトはシェル以外にも Python など様々な言語で書けるため、OS はこの 1 行目を見て「誰に実行させるか」を判断します。シェルスクリプトでは bash を指定するのが定番です。

なお、環境によって bash の置き場所が違う場合に備えて `#!/usr/bin/env bash` と書く流儀もあります。今は「1 行目で実行役を宣言する」という骨だけ押さえれば十分です。

## chmod +x — 実行権限を付けて動かす

書いただけのファイルはまだ「ただのテキスト」です。プログラムとして実行するには **実行権限** を付けます。

```bash
chmod +x hello.sh   # 実行権限を付与
./hello.sh          # 実行!
# Hello, script!
# 2026年 6月13日 土曜日 ...
```

実行時に `./` を付けるのは、第 3 章で見たとおり **カレントディレクトリが PATH に含まれていない** からです。「今ここにある `hello.sh`」と場所を明示して呼び出しています。逆に言えば、PATH に追加した `~/bin` にスクリプトを置けば、どこからでも `hello.sh` と名前だけで呼べる「自分のコマンド」になります。

## 注意点

- **`Permission denied` と言われたら、まず `chmod +x` を忘れていないか**を疑ってください。スクリプト作りでいちばん多いつまずきです。
- 拡張子 `.sh` は人間のための目印で、実行の仕組みには関係しません。実行役を決めるのはあくまで shebang です。
- `bash hello.sh` のように bash に直接渡せば、実行権限がなくても動きます。ただし道具として常用するなら、shebang + 実行権限を揃えるのが作法です。
- 前章のエイリアスはスクリプトの中では基本的に効きません。スクリプト内では省略せず本来のコマンドを書きます。

## まとめ

- シェルスクリプトはコマンドの並びを保存したファイル。書けば何度でも同じ手順を再生できる
- 1 行目の shebang(`#!/bin/bash`)が「どのプログラムで実行するか」を宣言する
- `chmod +x` で実行権限を付け、`./hello.sh` で実行。`~/bin` に置けば名前だけで呼べる
