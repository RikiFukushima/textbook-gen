## このセクションで学ぶこと

- `export` やエイリアスが消える問題を解決する「起動設定ファイル」の役割
- `.bashrc` / `.zshrc` がいつ読まれるか
- `source` で編集をその場で反映する方法

## 設定は毎回消える — だから起動時に読み直す

ここまでで `export PATH=...` のような設定を学びましたが、どれも **ターミナルを閉じれば消えます**。毎朝同じコマンドを打ち直すのは現実的ではありません。

シェルはこの問題を「**起動するたびに決まったファイルを読んで実行する**」という仕組みで解決しています。そのファイルが bash なら `~/.bashrc`、zsh なら `~/.zshrc` です。ここに設定コマンドを書いておけば、新しいターミナルを開くたびに自動で実行され、いつでも同じ環境が再現されます。

```mermaid
flowchart LR
    A["ターミナルを開く"] --> B["シェルが起動"]
    B --> C[".bashrc / .zshrc を上から実行"]
    C --> D["export や alias が効いた状態で<br>プロンプトが表示される"]
```

どちらのファイルを編集すべきかは、使っているシェルで決まります。

```bash
echo $SHELL   # /bin/zsh なら ~/.zshrc、/bin/bash なら ~/.bashrc
```

macOS の標準シェルは zsh、多くの Linux ディストリビューションの標準は bash です。読まれるタイミングは「**新しい対話シェルが起動するたび**」、つまりターミナルのウィンドウやタブを開くたびと考えてください。なお `~/.bash_profile` や `~/.zprofile` というログイン時専用のファイルもありますが、入門のうちは「日常の設定は `.bashrc` / `.zshrc` に書く」で困りません。

## 書いてみる — PATH とエディタの永続化

前のセクションまでの設定を永続化してみます。ファイルの末尾に追記するだけです。

```bash
# ~/.zshrc(bash なら ~/.bashrc)に追記する内容の例
export PATH="$HOME/bin:$PATH"
export EDITOR=vim
alias ll='ls -la'
```

編集しただけでは、いま開いているシェルには反映されません。読み込み直すには `source` を使います。

```bash
source ~/.zshrc   # 設定ファイルをその場で実行し直す
```

`source` は「指定したファイルの中身を、現在のシェルで実行する」コマンドです。新しいターミナルを開き直しても同じ効果が得られます。

## 注意点

- 設定ファイルの中身は「シェルが毎回実行するコマンド列」そのものです。**文法エラーが 1 行あると、以後すべての新しいターミナルでエラーが出ます**。編集したら必ず `source` で動作確認しましょう。
- **シェルとファイルの対応を間違えると何も起きません**。zsh を使っているのに `.bashrc` に書いても読まれないため、「設定したのに効かない」ときはまず `echo $SHELL` を確認します。
- 起動のたびに実行されるため、**重い処理を書くとターミナルを開くたびに待たされます**。設定ファイルは軽く保つのが快適さのコツです。

## まとめ

- `.bashrc` / `.zshrc` は新しい対話シェルが起動するたびに実行される設定ファイルで、export やエイリアスの定位置
- 自分のシェルは `echo $SHELL` で確認。対応しないファイルに書いても読まれない
- 編集後は `source ~/.zshrc` でその場に反映し、エラーが出ないか確かめる
