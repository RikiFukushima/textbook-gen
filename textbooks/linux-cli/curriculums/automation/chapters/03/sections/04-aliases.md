## このセクションで学ぶこと

- `alias` でよく打つコマンドに短い別名を付ける方法
- エイリアスの一覧表示と解除の方法
- 永続化の置き場所と、使ううえでの注意点

## alias — 自分専用の省略形を作る

毎日何十回も打つコマンドがあるなら、短い別名を付けてしまうのが **エイリアス** です。`alias 別名='本来のコマンド'` の形で定義します。

```bash
alias ll='ls -la'
alias gs='git status'

ll      # ls -la として実行される
gs      # git status として実行される
```

定義済みのエイリアスは引数なしの `alias` で一覧でき、`unalias` で解除できます。

```bash
alias        # いま定義されているエイリアスを一覧表示
unalias ll   # ll を解除
```

実務でよく見る定番をいくつか挙げます。長いオプションの組み合わせや、打ち間違えやすい長いコマンドほどエイリアスの効果が大きくなります。

| エイリアス | 中身 | ねらい |
| --- | --- | --- |
| `ll` | `ls -la` | 詳細一覧を 2 文字で |
| `gs` | `git status` | 1 日に何十回も打つ確認を短縮 |
| `..` | `cd ..` | 1 つ上の階層へ素早く移動 |

定義したエイリアスはそのシェル限りで、ターミナルを閉じると消えます。気に入ったものは前のセクションのとおり `.bashrc` / `.zshrc` に書いて永続化しましょう。設定ファイルが「自分の道具箱」に育っていきます。

## 注意点

- **既存のコマンド名を上書きするエイリアスには注意**が必要です。たとえば `alias rm='rm -i'`(削除前に毎回確認)は安全策として定番ですが、これに慣れきると、エイリアスのない他人のサーバーで素の `rm` を打って事故につながります。上書き型は「保険」程度に考え、頼りすぎないことです。
- エイリアスを一時的に無視して本来のコマンドを使いたいときは、`\rm` のように **先頭にバックスラッシュ** を付けます。
- エイリアスは対話シェル用の道具です。**シェルスクリプトの中では基本的に効きません**。次章で書くスクリプトでは、省略せず本来のコマンドを書きます。

## まとめ

- `alias 別名='コマンド'` で省略形を定義。`alias` で一覧、`unalias` で解除
- 永続化したいものは `.bashrc` / `.zshrc` に書く
- 既存コマンドの上書きは事故のもとになりうる。スクリプト内では効かない点も覚えておく
