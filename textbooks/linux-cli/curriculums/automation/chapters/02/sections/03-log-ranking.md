## このセクションで学ぶこと

- 前のセクションのフィルタを組み合わせ、アクセスログから「ランキング」を作る
- パイプラインを 1 段ずつ足しながら組み立てる手順を身につける
- `uniq -c | sort -nr` という「多い順集計」の慣用句を覚える

## 題材 — 404 を多発させているアクセス元はどこか

Web サーバーのアクセスログ `access.log` を題材にします。各行の先頭はアクセス元の IP アドレス、末尾はステータスコードです。

```bash
head -3 access.log
# 192.168.1.20 - - [12/Jun/2026:10:03:21] "GET /api/users HTTP/1.1" 200
# 203.0.113.5 - - [12/Jun/2026:10:03:22] "GET /admin HTTP/1.1" 404
# 192.168.1.34 - - [12/Jun/2026:10:03:25] "GET /index.html HTTP/1.1" 200
```

知りたいのは「**404(存在しないページへのアクセス)を多く出しているアクセス元トップ 3**」。怪しい探索行動の調査などで実際によくある分析です。これを 1 段ずつ組み立てます。

まず `grep` で 404 の行だけ通します。

```bash
grep " 404" access.log
```

次に、各行から IP アドレスだけ切り出します。ここで助っ人として `cut` を 1 つ覚えてください。`cut -d " " -f 1` で「スペース区切りの 1 列目」を切り出せます。

```bash
grep " 404" access.log | cut -d " " -f 1
# 203.0.113.5
# 198.51.100.7
# 203.0.113.5
# ...
```

あとは前のセクションの慣用句、`sort | uniq -c` で集計し、回数の多い順に並べ替えて上位だけ表示します。

```bash
grep " 404" access.log | cut -d " " -f 1 | sort | uniq -c | sort -nr | head -3
#  42 203.0.113.5
#  17 198.51.100.7
#   3 192.168.1.20
```

データが各段でどう姿を変えるかを図にすると、こうなります。

```mermaid
flowchart LR
    A["access.log<br>(全行)"] --> B["grep ' 404'<br>404 の行だけ"]
    B --> C["cut -d ' ' -f 1<br>IP だけに"]
    C --> D["sort<br>同じ IP を隣に"]
    D --> E["uniq -c<br>回数を集計"]
    E --> F["sort -nr<br>多い順に"]
    F --> G["head -3<br>上位 3 件"]
```

それぞれは単機能のフィルタなのに、つなぐだけで「ログ分析ツール」が完成しました。2 回目の `sort` に付けた `-nr` は、`-n`(数値として比較)と `-r`(逆順=大きい順)の組み合わせで、ランキング作りの定番です。

## 注意点 — いきなり全部書かない

長いパイプラインを一気に書くと、結果がおかしいときにどの段が原因か分かりません。コツは、**1 段足すたびに実行して途中結果を目で確認する**ことです。今回の例も `grep` だけ → `cut` を足す → `sort | uniq -c` を足す、と確認しながら育てました。慣れた人ほどこの「小刻みな確認」を守っています。

もう 1 つ、`sort -nr` の `-n` を忘れると数字が文字として比較され、`9` が `42` より大きい(先頭の文字 `9` > `4`)と判定されて順位が狂います。集計値の並べ替えには必ず `-n` を付けてください。

## まとめ

- `grep → cut → sort → uniq -c → sort -nr → head` で、ログからランキングが作れる
- `uniq -c | sort -nr` は「多い順集計」の慣用句。`-n` を忘れると数字が文字として比較され順位が狂う
- パイプラインは 1 段ずつ足し、途中結果を確認しながら組み立てる
