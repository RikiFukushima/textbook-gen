## このセクションで学ぶこと

- LangSmith が LangChain / LangGraph と密結合した観測性プラットフォームであることを理解する
- マネージド SaaS としてのメリット(導入の速さ、ダッシュボード、評価機能)を把握する
- データが外部に出るというトレードオフを認識し、選定の判断軸を持つ

## LangSmith とは

LangSmith は、LangChain 社が提供する LLM アプリ向けの観測性・評価プラットフォームです。LangChain / LangGraph と同じエコシステムで開発されており、これらのフレームワークで書かれたアプリの内部処理を、ほぼ追加コードなしでトレースできます。環境変数を 2-3 個セットするだけで有効化できる手軽さが大きな魅力です。

LangSmith の設計思想は、「観測 → 評価 → 改善のループを 1 つのプラットフォームで閉じる」ことにあります。トレースを眺めて気になる挙動を見つけたら、その Run をそのまま Dataset に追加し、改善後のバージョンで回帰評価を回す、という流れが UI 上で完結します。前章で扱った評価とこの章の観測性が、運用上は地続きであることが思想に表れています。

## 主要機能

LangSmith の機能は大きく次の 4 つです。

- トレース可視化: Run のツリー、各 Run の入出力・レイテンシ・トークン数・コストを UI で確認
- データセット管理: トレースから直接データセットを構築し、評価実行に流用
- 評価 (Evaluator): LLM-as-a-Judge やカスタム関数を Dataset に対して実行
- プロンプト管理: プロンプトをバージョン管理し、本番のトレースと紐づけて追跡

LangGraph のアプリでは、Conditional Edge の分岐や各ノードの状態遷移もそのままトレースに現れます。「どのノードでどの判断をしたか」をクリックで辿れるため、Agent のデバッグ体験が大きく変わります。

## 具体例: 「遅い」を切り分ける

「Agent の応答が遅い」というよくある不満を考えます。LangSmith のトレースを開けば、Run ツリー上で各ステップのレイテンシが横棒として描かれており、リトリーバルに 3 秒、LLM 呼び出しに 5 秒、ツール実行に 2 秒、といった内訳が一目でわかります。

「リトリーバルが支配的なら検索インデックスを見直す」「LLM 呼び出しが長いなら streaming や軽量モデルを検討する」と、次の手の根拠もそこで得られます。これは print デバッグでは得られない解像度です。

## 注意点: データが外部に出るトレードオフ

最大のトレードオフは、トレースデータが LangSmith のサーバに送られることです。ユーザー入力・検索結果・LLM 出力には、PII や社内秘情報が含まれる可能性があります。社内データを扱う教育サービスや BtoB ツールでは、契約条件・データ取扱いポリシー・準拠する地域(リージョン)を確認したうえで判断する必要があります。

セルフホスト版も提供されていますが、運用負荷が一気に上がるため「マネージドの手軽さ」というメリットは相殺されます。次の節で Langfuse と並べて、選定の軸を整理します。

## まとめ

- LangSmith は LangChain / LangGraph と密結合した観測性 SaaS
- 観測 → 評価 → 改善のループを 1 プラットフォームで閉じる思想
- データが外部に出るため、扱うデータ性質と契約条件で採否を判断する
