## このセクションで学ぶこと

- 設計書レビューを 5 カテゴリで自己点検できる
- チェック項目を「他人にレビューしてもらう前の準備」として使える
- 設計書を書く目的「合意とリスク低減」に立ち戻れる

## チェックリストは「他人に見せる前の準備」

設計書ができたら、いきなりレビュー会に出すのではなく、まず自分で点検する時間を取ります。**自己レビューを挟むかどうかで、レビュー会の議論の質が大きく変わります**。本書では次の 5 カテゴリ・合計 20 項目のチェックリストを推奨します。これは AI プロジェクトのロードマップを学んだ読者が、自分の設計書を「合意とリスク低減」の道具として機能させるための最終フィルタです。

### カテゴリ 1: 要件(4 項目)

- [ ] 解くべき問題が「誰が・どの業務で・どれくらいの時間/コスト」を消費しているか、数字で書かれているか
- [ ] 成功基準が定量メトリクスと定性的な反応の両方で書かれているか
- [ ] 非要件(やらないこと)が明示されているか
- [ ] 制約(コスト・期限・セキュリティ・PII の扱い)が列挙されているか

### カテゴリ 2: アーキテクチャ(4 項目)

- [ ] ユースケース図とコンポーネント図が分かれているか(1 枚に詰め込んでいないか)
- [ ] コンポーネント図で Agent / RAG / 既存基盤の責任範囲が subgraph などで明示されているか
- [ ] バッチ処理・観測性など本流でない流れが線種で区別されているか
- [ ] 図に実装詳細(モデル名・バージョン)を書き込みすぎていないか(別表に逃がしているか)

### カテゴリ 3: 評価(4 項目)

- [ ] オフラインメトリクスとオンラインメトリクスの両方が定義されているか
- [ ] 各メトリクスに合格ラインの数値が書かれているか
- [ ] ゴールデンセットの規模(30-50 件目安)と作り方が書かれているか
- [ ] リリース前バッチ評価と運用サンプリング評価の頻度が分けて書かれているか

### カテゴリ 4: 運用(4 項目)

- [ ] 段階的リリースのフェーズ定義(内部 → 限定 → 本番)があるか
- [ ] 各フェーズの「移行条件」が数値の閾値で書かれているか
- [ ] 想定リスクが「リスク・軽減策・検知方法」の 3 点セットで書かれているか
- [ ] 観測性(トレース・ダッシュボード・アラート)の設計が含まれているか

### カテゴリ 5: セキュリティ(4 項目)

- [ ] PII が含まれるデータ経路(取り込み・ログ・外部 API)の 3 点が洗い出されているか
- [ ] プロンプトインジェクション対策(出力フィルタ・権限分離)が記載されているか
- [ ] 認証・認可(誰が何にアクセスできるか)が設計されているか
- [ ] インシデント発生時の連絡経路と緊急停止手順が書かれているか

## 使い方 — 「読み手ごと」に再点検する

このチェックリストは自己レビューの土台です。さらに精度を上げるなら、**読み手の視点に立って読み返す** ことを推奨します。

- **経営層が読む** つもりで読み返す:ユースケース図とコスト・期限がすぐ見つかるか?
- **業務オーナーが読む** つもりで読み返す:現場のペインとどう解消されるかが分かるか?
- **エンジニアが読む** つもりで読み返す:アーキ図とリスクの軽減策で実装範囲がイメージできるか?
- **運用担当が読む** つもりで読み返す:アラート・移行条件・緊急停止手順があるか?

それぞれの読み手が「自分の関心事の答え」を 3 分以内に見つけられない箇所は、書き直すか、参照表を追加します。

## 注意点 — チェックリストを「埋める儀式」にしない

最後の警告です。チェックリストは項目を全部チェックすれば良いものではありません。**「なぜこの項目が必要だったか」を思い出しながら埋める** ことが本質です。たとえば「移行条件が数値で書かれているか」は、本番リリースを止める権限を誰かが持つために書きます。数値を書いたけれど判断者が決まっていない、では機能しません。

そして設計書全体を読み終えたら、最初に立ち戻ってください。**設計書は「書くため」に書くのではなく、合意とリスク低減のために書く**。チェックリストの最後の問いは常にこうです。「この設計書を関係者全員が読んだとき、本当に同じ絵が見えているか?」

## まとめ

- 自己レビューは「要件・アーキ・評価・運用・セキュリティ」の 5 カテゴリ 20 項目でチェックする
- 読み手(経営・業務・エンジニア・運用)ごとに再点検すると抜けが見える
- チェックリストは儀式ではなく、合意とリスク低減のための道具として使う
