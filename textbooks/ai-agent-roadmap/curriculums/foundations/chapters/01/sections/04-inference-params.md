# 推論パラメータの実践

## このセクションで学ぶこと

- 出力が確率分布からのサンプリングであること
- temperature / top_p / top_k がそれぞれ何を制御するか
- タスクに応じたパラメータの使い分け

## 出力は「選び方」で変わる

前のセクションのとおり、LLM は次トークンの **確率分布** を出します。実際の出力は、その分布から 1 つを選ぶ(サンプリングする)ことで決まります。この「選び方」を制御するのが推論パラメータです。

- **temperature**: 分布の尖り具合を変えます。低い(0 に近い)ほど最も確率の高いトークンに偏り、決定的で安定した出力に。高いほど低確率のトークンも選ばれやすく、多様で創造的(かつ不安定)になります。
- **top_p(核サンプリング)**: 確率の高い順に足していき、累積が p(例: 0.9)に達するまでのトークンだけを候補にします。
- **top_k**: 単純に上位 k 個だけを候補に残します。

```python
client.responses.create(
    model="...",
    input="...",
    temperature=0.2,   # 事実重視なら低め
    top_p=0.9,
)
```

## タスクに応じた使い分け

事実を正確に回答したい・抽出や分類など「ブレてほしくない」タスクでは **temperature を低く**(例: 0〜0.3)します。アイデア出しやキャッチコピー生成など多様性が欲しい場合は **高め**(例: 0.7〜1.0)にします。top_p / top_k は「極端に的外れな低確率トークン」を切り捨てて暴走を防ぐ役割で、temperature と組み合わせて使います。

たとえば「カテゴリのどれかに分類する」タスクで temperature を高くすると、本来選ばれないはずのカテゴリがたまに混ざり、結果が不安定になります。逆に小説の続きを書かせるのに temperature を 0 にすると、当たり障りのない平板な文章になりがちです。タスクが「正確さ」を求めるのか「多様さ」を求めるのかで方向を決めるのが基本です。

## 実務での注意点

temperature を 0 にしても、出力が完全に固定されるとは限りません(実装やハードウェアの都合で揺れることがあります)。また temperature と top_p を両方とも強くいじると効果が干渉し、挙動が読みにくくなります。まずは片方を動かして検証するのが安全です。再現性が重要な評価では、パラメータを固定して記録しておきましょう。

## まとめ

- LLM の出力は確率分布からのサンプリングで決まる
- temperature は多様性、top_p / top_k は候補の絞り込みを制御する
- 事実系は低 temperature、創造系は高 temperature。いじるパラメータは絞る
