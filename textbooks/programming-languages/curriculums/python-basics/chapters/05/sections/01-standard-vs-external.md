## このセクションで学ぶこと

- ライブラリとは何かを説明できる
- 標準ライブラリと外部ライブラリの違いを理解する
- 外部ライブラリを使うには導入作業が必要だと分かる

## ライブラリとは「他人が書いた便利な部品」

プログラムを書いていると、日付の計算や JSON の読み書き、Web へのアクセスなど、多くの人が共通して必要とする処理に何度も出会います。こうした機能を毎回ゼロから書くのは非効率です。そこで、よく使う機能をまとめて再利用できるようにしたコードの集まりが **ライブラリ** です。ライブラリを使えば、自分は本当に作りたい部分だけに集中できます。

第 3 章で学んだ `import` は、まさにこのライブラリを呼び出すための仕組みでした。Python のライブラリは大きく二種類に分かれます。**標準ライブラリ** と **外部ライブラリ** です。

## 標準ライブラリ:最初から入っている

**標準ライブラリ** は、Python をインストールした時点で一緒に付属してくるライブラリ群です。追加の導入作業なしに、いきなり `import` するだけで使えます。

```python
import math
import datetime

print(math.sqrt(2))          # 平方根
print(datetime.date.today()) # 今日の日付
```

`math`(数学)、`datetime`(日付)、`json`(JSON 処理)、`os`(OS 操作)、`random`(乱数)など、数百のモジュールが標準で揃っています。「まず標準ライブラリで足りないか」を確認するのが Python の基本姿勢です。

## 外部ライブラリ:あとから追加する

一方、世の中には Python 本体には含まれていない、第三者が開発・公開したライブラリが膨大に存在します。これが **外部ライブラリ** です。たとえば Web アクセスを簡単にする `requests`、表形式データを扱う `pandas`、グラフを描く `matplotlib` などが代表例です。

これらは標準ライブラリと違い、最初からは入っていません。使うには **PyPI**(Python Package Index)という公式の配布サイトから取得してインストールする必要があります。インストールしていない外部ライブラリを `import` しようとすると、次のようなエラーになります。

```python
import requests
# ModuleNotFoundError: No module named 'requests'
```

このエラーは「そんなモジュールは見つからない」という意味で、初学者がよく出会います。多くの場合、原因は外部ライブラリの導入忘れです。次のセクションで学ぶ `pip` を使えば、PyPI から目的のライブラリを簡単にインストールできます。

## 注意点:標準か外部かを意識する

コードを読むときは、`import` されているのが標準ライブラリなのか外部ライブラリなのかを意識しましょう。標準ライブラリならどの環境でも動きますが、外部ライブラリは導入されていない環境では動きません。他の人にコードを渡すとき、この違いを伝えていないと「自分の環境では動かない」というトラブルにつながります。

## まとめ

- ライブラリはよく使う機能をまとめた再利用可能な部品で、`import` で呼び出す。
- 標準ライブラリは最初から付属し、外部ライブラリは PyPI から別途インストールが必要。
- 外部ライブラリの導入忘れは `ModuleNotFoundError` の典型的な原因。
