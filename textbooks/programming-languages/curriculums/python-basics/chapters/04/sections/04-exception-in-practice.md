## このセクションで学ぶこと

- 例外は「握りつぶす」のではなく適切に対処・記録するという基本姿勢を理解する
- ファイル読み込みやユーザー入力など、実務でよくある例外パターンを書ける
- `raise` で自分から例外を出す使いどころを理解する

## まず「握りつぶさない」 ― 例外処理の基本姿勢

前のセクションで `try` / `except` の書き方を学びました。実務では「書ける」だけでなく「どう使うか」が問われます。もっとも大切な原則は、**例外を握りつぶさない** ことです。

握りつぶしとは、例外を捕まえたのに何もしない、次のようなコードを指します。

```python
try:
    risky_process()
except Exception:
    pass  # 何が起きても無視 ← 危険
```

これをやると、エラーが起きても表面上は正常に動いているように見え、後から「なぜか結果がおかしい」「データが欠けている」といった原因不明のトラブルにつながります。例外を捕まえたら、最低限 **何が起きたかを記録(ログ出力)** し、必要なら処理をやり直すか、安全に中断します。

## 実務でよくある例外パターン

実際の現場で頻出するのは、次のような「外部に依存する処理」での例外です。自分のコードの外(ファイル・ユーザー・ネットワーク)が絡む部分は、いつ失敗してもおかしくないと考えて備えます。

```python
# パターン 1: 設定ファイルが無くてもデフォルトで動かす
try:
    with open("config.txt", "r", encoding="utf-8") as f:
        config = f.read()
except FileNotFoundError:
    print("設定ファイルが無いので初期設定で起動します")
    config = "default"
```

ファイルが無いという例外(`FileNotFoundError`)を捕まえ、止まる代わりにデフォルト値で続行しています。「失敗したらどうリカバリーするか」をセットで考えるのがコツです。

```python
# パターン 2: ユーザー入力を正しく受け取るまで繰り返す
while True:
    try:
        age = int(input("年齢を入力: "))
        break                       # 変換できたらループを抜ける
    except ValueError:
        print("数値を入力してください")
```

入力という「人間が絡む処理」は失敗が前提です。`try` をループの中に置き、正しい入力が来るまでやり直しています。

## raise ― 自分から例外を出す

例外は捕まえるだけでなく、**自分から出す** こともできます。関数に不正な値が渡されたとき、間違ったまま処理を進めるより、その場で `raise` して呼び出し元に知らせるほうが安全です。

```python
def set_age(age):
    if age < 0:
        raise ValueError("年齢に負の数は指定できません")
    return age
```

こうしておくと、呼び出し側は `try` / `except` で対処できますし、放置すればプログラムが止まって問題に早く気づけます。「おかしな値を黙って受け入れない」ことが、バグを早期に発見する近道です。

## 注意点

- 捕まえる例外は **具体的な種類に絞る**(`except FileNotFoundError:` など)。`except Exception:` で何でも捕まえると、想定外のバグまで隠してしまいます。
- 例外を捕まえたら、`print(e)` やログでメッセージを残し、後から原因を追えるようにします。
- リカバリーできない例外は、無理に握りつぶさず止める判断も大切です。止まること自体が「異常を知らせるサイン」になります。

## まとめ

- 例外は握りつぶさず、記録して対処するか、安全に中断するのが基本姿勢。
- ファイル・ユーザー入力など外部に依存する処理は失敗前提で `try` / `except` を備える。
- `raise` で不正な値を自分から知らせると、バグに早く気づける。
