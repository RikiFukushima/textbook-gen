## このセクションで学ぶこと

- デフォルト引数で「省略できる引数」を用意できる
- キーワード引数で引数を名前付きで渡せる
- 位置引数とキーワード引数の使い分けと注意点を理解する

## デフォルト引数 ― 省略できる引数を用意する

前のセクションでは、関数を呼ぶときに引数を必ず渡していました。しかし「ほとんどの場合は同じ値でいいけれど、たまに変えたい」という引数もあります。そんなときに使うのが **デフォルト引数** です。関数定義の段階で `引数名=初期値` と書いておくと、呼び出し時にその引数を省略できます。

```python
def greet(name, greeting="こんにちは"):
    print(f"{greeting}、{name}さん!")

greet("田中")                  # こんにちは、田中さん!
greet("鈴木", "おはよう")       # おはよう、鈴木さん!
```

`greeting="こんにちは"` がデフォルト引数です。省略すれば `"こんにちは"` が使われ、明示すればその値が使われます。よく使う設定値を初期値にしておくと、呼び出し側がシンプルになります。

デフォルト値を持つ引数は、**持たない引数より後ろに書く**というルールがあります。`def greet(greeting="こんにちは", name)` のように逆にすると、エラーになります。

## キーワード引数 ― 名前を指定して渡す

これまでの引数の渡し方は、**書いた順番**で対応づけられていました。これを **位置引数** と呼びます。一方、`引数名=値` の形で名前を指定して渡す方法を **キーワード引数** と呼びます。

```python
def make_user(name, age, city):
    print(f"{name}({age}歳, {city})")

make_user("田中", 30, "東京")              # 位置引数
make_user(name="田中", city="東京", age=30)  # キーワード引数(順序自由)
```

キーワード引数を使うと、**順序を気にせず**渡せます。引数が多い関数では、`make_user("田中", 30, "東京")` だけを見ても 30 や "東京" が何を意味するか分かりにくいですが、`age=30, city="東京"` と書けば一目瞭然です。可読性が大きく上がります。

## 位置引数とキーワード引数を混ぜる

位置引数とキーワード引数は混ぜて使えますが、**位置引数を先に、キーワード引数を後に**書く必要があります。

```python
make_user("田中", age=30, city="東京")   # OK
# make_user(name="田中", 30, "東京")     # エラー(キーワードの後に位置引数は不可)
```

## 注意点 ― 可変オブジェクトをデフォルト値にしない

デフォルト引数には、よく知られた落とし穴があります。**リストや辞書のような「変更できるオブジェクト」をデフォルト値にしない**ことです。

```python
def add_item(item, items=[]):   # 危険なパターン
    items.append(item)
    return items

print(add_item("A"))   # ['A']
print(add_item("B"))   # ['A', 'B'] ← 前回の結果が残ってしまう!
```

デフォルト値のリストは関数定義時に一度だけ作られ、呼び出しごとに共有されるため、意図しない蓄積が起きます。回避策は、デフォルトを `None` にして関数内で新しく作ることです。

```python
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

## まとめ

- デフォルト引数(`引数名=初期値`)で、省略可能な引数を用意できる。
- キーワード引数(`引数名=値`)は順序に依存せず、可読性を高める。
- デフォルト値にリストや辞書を使わない ― `None` を使って関数内で作る。
