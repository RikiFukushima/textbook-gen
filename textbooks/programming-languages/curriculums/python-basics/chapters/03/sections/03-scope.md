## このセクションで学ぶこと

- ローカル変数とグローバル変数の違いを理解する
- 関数内で作った変数が外から見えない理由を説明できる
- global の使いどころと、避けるべき理由を押さえる

## スコープとは ― 変数の見える範囲

関数を使い始めると、「関数の中で作った変数が、関数の外で使えない」という現象に出会います。これは Python のバグではなく、**スコープ**という仕組みによるものです。スコープとは、**変数が有効で参照できる範囲**のことです。変数には「見える範囲」が決まっている、と考えてください。

スコープがあるおかげで、関数ごとに同じ名前の変数を安心して使えます。もしすべての変数がどこからでも見えると、別々の関数で偶然同じ名前を使っただけで値が壊れてしまいます。スコープは、変数同士の衝突を防ぐ仕切りの役目を果たしています。

## ローカル変数 ― 関数の中だけの変数

関数の中で作った変数を **ローカル変数** と呼びます。ローカル変数は、その関数の中だけで生き、関数が終わると消えます。外からは見えません。

```python
def calc():
    total = 100      # ローカル変数
    print(total)     # 100

calc()
print(total)         # エラー: total は外からは見えない
```

`total` は `calc` の中で作られたローカル変数なので、関数の外で `print(total)` しようとするとエラーになります。関数の引数も、同じくローカル変数の一種です。

## グローバル変数 ― 全体から見える変数

一方、関数の外で定義した変数を **グローバル変数** と呼びます。グローバル変数は、プログラム全体から参照できます。関数の中からも「読む」だけなら問題なく見えます。

```python
tax_rate = 0.1            # グローバル変数

def price_with_tax(price):
    return price * (1 + tax_rate)   # 外の tax_rate を読める

print(price_with_tax(1000))   # 1100.0
```

このように、設定値のような「全体で共有したい値」をグローバル変数にしておくと、各関数から参照できて便利です。

## 注意点 ― 関数内からの書き換えと global

ここで注意したいのが、**関数の中からグローバル変数を「書き換え」ようとしたとき**の挙動です。関数内で同じ名前に代入すると、Python はそれを新しいローカル変数とみなし、外側のグローバル変数は変わりません。

```python
count = 0

def increment():
    count = count + 1   # エラー: count はローカル扱いになる

increment()
```

どうしても関数内からグローバル変数を書き換えたい場合は、`global count` と宣言します。

```python
count = 0

def increment():
    global count
    count = count + 1

increment()
print(count)   # 1
```

ただし、`global` の多用は **どこで値が変わったか追いにくくなる**ため、実務では避けるのが基本です。値を変えたいときは「引数で受け取り、`return` で返す」という前のセクションの形を使うほうが、影響範囲が明確で安全です。

## まとめ

- スコープは変数の見える範囲。関数内のローカル変数は外から見えない。
- グローバル変数は読むだけなら関数内からも参照できる。
- 書き換えは `global` で可能だが追いにくくなる ― 引数と return を優先する。
