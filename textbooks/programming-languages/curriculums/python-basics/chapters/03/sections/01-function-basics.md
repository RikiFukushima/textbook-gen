## このセクションで学ぶこと

- def を使って関数を定義し、呼び出せるようになる
- 引数で関数に値を渡し、戻り値で結果を受け取れる
- 関数を使って処理を「名前のついた部品」にまとめる意味を理解する

## 関数とは ― 処理に名前を付けた部品

プログラムを書いていると、「同じような処理を何度も書く」場面が必ず出てきます。たとえば「2 つの数を足して結果を表示する」という処理を、コードのあちこちにコピーして書くと、修正のたびにすべての箇所を直す羽目になります。

そこで使うのが **関数(function)** です。関数とは、**ひとまとまりの処理に名前を付けて、何度でも呼び出せるようにした部品**のことです。一度定義しておけば、その名前を呼ぶだけで中身の処理がまるごと実行されます。

Python では `def` というキーワードで関数を定義します。`def` は define(定義する)の略です。

```python
def greet():
    print("こんにちは!")

greet()  # こんにちは!
greet()  # こんにちは!(何度でも呼べる)
```

`def greet():` の行で「greet という名前の関数を作る」と宣言し、インデント(字下げ)された部分が関数の中身(本体)です。`greet()` と書くと、その中身が実行されます。インデントが関数のまとまりを表す点は、前章で学んだ `if` や `for` と同じ考え方です。

## 引数 ― 関数に値を渡す

関数は、毎回まったく同じことをするだけでは応用が利きません。呼び出すたびに「違う値」で動いてほしいことがほとんどです。そのために **引数(ひきすう)** を使います。引数は、関数を呼び出すときに外から渡す値です。

```python
def greet(name):
    print(f"こんにちは、{name}さん!")

greet("田中")   # こんにちは、田中さん!
greet("鈴木")   # こんにちは、鈴木さん!
```

`def greet(name):` のカッコの中の `name` が引数(正確には仮引数=パラメータ)です。呼び出し側で `greet("田中")` と書くと、`"田中"` が `name` に入った状態で関数が動きます。引数はカンマで区切って複数渡せます。

```python
def add(a, b):
    print(a + b)

add(3, 5)   # 8
```

## 戻り値 ― 結果を受け取る

`print` で表示するだけなら上の例で十分ですが、多くの場合は「関数が計算した結果を、呼び出し元で使いたい」はずです。そこで使うのが **戻り値(返り値)** です。`return` 文で値を返すと、関数の呼び出し部分がその値に置き換わります。

```python
def add(a, b):
    return a + b

result = add(3, 5)   # result に 8 が入る
print(result * 2)    # 16(返ってきた値を再利用できる)
```

`return` を実行すると、その時点で関数は終了し、値を呼び出し元へ返します。`return` を書かない関数は、自動的に `None`(値がないことを表す特別な値)を返します。

## 注意点 ― print と return は別物

初学者がつまずきやすいのが、**`print` と `return` の違い**です。`print` は画面に表示するだけで、値を呼び出し元に渡しません。一方 `return` は値を呼び出し元に渡しますが、画面には何も表示しません。

```python
def add_print(a, b):
    print(a + b)      # 表示するだけ

x = add_print(3, 5)   # 8 が表示される。でも x は None
```

「計算結果を後で使いたい」なら `return`、「ただ表示したい」なら `print` と覚えてください。実務では、関数は `return` で結果を返し、表示は呼び出し側で行うのが基本形です。こうすると関数を別の処理にも使い回しやすくなります。

## まとめ

- 関数は `def` で定義し、ひとまとまりの処理に名前を付けた再利用できる部品。
- 引数で値を渡し、`return` で結果(戻り値)を受け取る。
- `print` は表示するだけ、`return` は値を返す ― この違いを区別する。
