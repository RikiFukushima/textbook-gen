## このセクションで学ぶこと

- Python がインタプリタ型で書きやすい言語であることを説明できる
- ライブラリの豊富さが Python の強みである理由を理解する
- Python の得意分野(分析・AI・自動化)と不得意分野を把握する

## Python とはどんな言語か

**Python**(パイソン)は、**読みやすさ・書きやすさ** を最優先に設計された言語です。第 1 章の軸でいえば「インタプリタ型」「動的型付け」にあたり、Go とはちょうど対照的な性質を持ちます。事前に型を書かなくてよく、コンパイルの手順も挟まないため、思いついたコードをすぐ試せるのが大きな魅力です。

Python の特徴は、次の 3 点にまとめられます。

- **書きやすい**: 文法が英語に近く、少ない行数で意図を表現できます。プログラミングを初めて学ぶ人にも勧められる言語です。
- **動的型付けで手軽**: 変数の型を宣言せずに使えるため、短いコードを手早く書けます。
- **ライブラリが豊富**: データ分析・AI・Web・自動化など、あらゆる分野に **ライブラリ**(再利用できる部品集)が揃っています。これが Python 最大の武器です。

## 具体例: ライブラリで「車輪の再発明」を避ける

Python では、高度な処理も「すでに誰かが作った部品」を読み込むだけで使えることがよくあります。たとえば表計算的なデータ処理は、`pandas` というライブラリを読み込めば数行で書けます。

```python
import pandas as pd

df = pd.read_csv("sales.csv")   # CSV を読み込む
print(df["amount"].sum())       # 合計を計算する
```

自分でゼロから集計処理を書く必要はありません。データ分析なら `pandas`、AI・機械学習なら `PyTorch` や `scikit-learn`、業務の自動化ならファイル操作やスクレイピングのライブラリ、というように **「やりたいこと」にほぼ必ず対応する部品がある** のが Python の世界です。とくに AI・機械学習の分野では事実上の標準言語になっており、研究から実務まで幅広く使われています。

## 注意点

書きやすさと引き換えに、Python は **実行速度が遅め** です。インタプリタ型のため、Go のようなコンパイル型に比べると同じ計算でも時間がかかります。大量のデータを高速に処理したい場面では、速さが効く部分だけ別言語に任せる、といった工夫がされることもあります。

また動的型付けは手軽な反面、型に関する間違いが **実行してみるまで気づきにくい** という弱点があります。プログラムが大きくなるほどこの弱点は効いてくるため、次のセクションでは同じ処理を Go と Python で比べ、両者の違いを具体的に見ていきます。

## まとめ

- Python はインタプリタ型・動的型付けで、読みやすく書きやすい言語。
- 豊富なライブラリが最大の強みで、分析・AI・自動化で広く使われる。
- 実行は遅め、型の間違いに気づきにくい点が不得意分野。
