## このセクションで学ぶこと

- Go がコンパイル型で実行が速い言語であることを説明できる
- 文法がシンプルでチーム開発に向いている理由を理解する
- Go の得意分野(サーバ・ツール)と不得意分野を把握する

## Go とはどんな言語か

**Go**(ゴー、Golang とも呼ばれます)は、Google が開発したプログラミング言語です。第 1 章で学んだ「コンパイル型」「静的型付け」という性質を持ち、その上で **文法をとことんシンプルに保つ** ことを設計の柱にしています。「速く動き、読みやすく、大人数で書いても破綻しにくい言語」を目指して作られた、と理解しておくとよいでしょう。

Go の特徴は、大きく次の 3 点に整理できます。

- **実行が速い**: ソースコードを機械語にコンパイルしてから動かすため、実行速度が速く、サーバのように負荷のかかる用途に向きます。
- **文法がシンプル**: 言語仕様が小さく、書き方のバリエーションが少なめです。誰が書いても似たコードになりやすく、チームでの読み書きが楽になります。
- **配布が簡単**: コンパイル結果を **単一バイナリ**(ひとつの実行ファイル)にまとめられます。そのファイルを置くだけで動くため、配布や運用が手軽です。

## 具体例: 並行処理とサーバ用途

Go がとくに得意とするのが、**たくさんの処理を同時にさばく** 場面です。Go には **ゴルーチン** という軽量な並行処理の仕組みが標準で用意されていて、`go` というキーワードを付けるだけで処理を並行して走らせられます。

```go
// 「hello」を別の流れで並行に実行する
go sayHello()
```

たとえば Web サーバは、多数の利用者からのリクエストを同時に受け付ける必要があります。Go はこうした「同時に大量にさばく」処理を書きやすく、しかも速いため、API サーバやマイクロサービスの実装言語として広く採用されています。Docker や Kubernetes といった有名なインフラ向けツールも Go で書かれており、コマンドラインツールづくりでも定番の選択肢です。

## 注意点

一方で、Go にも不得意な領域があります。文法をシンプルに保つ方針の裏返しとして、便利だが複雑な書き方をあえて省いている部分があり、短いコードでサッと書きたい用途では記述量が多く感じられることがあります。

また、データ分析や機械学習のように **既製のライブラリが充実していること** が決め手になる分野では、後で学ぶ Python に分があります。Go は「速さと運用のしやすさ」が効く領域で力を発揮する言語であり、何でも一番得意なわけではない、という位置づけを押さえておきましょう。

## まとめ

- Go はコンパイル型・静的型付けで、実行が速く配布も簡単な言語。
- 文法がシンプルで、並行処理が得意。サーバやツールづくりに向く。
- 短く書く用途やライブラリ重視の分野は不得意で、万能ではない。
