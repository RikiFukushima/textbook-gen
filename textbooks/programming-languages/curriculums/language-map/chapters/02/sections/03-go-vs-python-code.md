## このセクションで学ぶこと

- 同じ処理を Go と Python で書いたコードの違いを読み取れる
- 型宣言とエラー処理の有無による記述量の差を理解する
- コードの見た目から両言語の設計思想を読み取れる

## 同じお題を二つの言語で

ここまで Go と Python の性質を言葉で説明してきました。実際のコードを並べると、その違いはもっとはっきり見えます。お題は **「数値のリストを受け取り、その合計を返す」** という単純な処理です。細かい文法を覚える必要はありません。**全体の雰囲気と長さの違い** を感じ取ってください。

まず Python です。

```python
def total(numbers):
    return sum(numbers)

print(total([1, 2, 3]))   # 6
```

次に Go です。

```go
func total(numbers []int) int {
    sum := 0
    for _, n := range numbers {
        sum += n
    }
    return sum
}
```

## 具体例: どこが違うかを読み取る

二つを見比べると、設計思想の違いが浮かび上がります。

- **型宣言**: Python は `numbers` が何の型かを書いていません。Go は `numbers []int`(整数のリスト)、戻り値 `int` と **型宣言** を明示します。Go は事前に型を確定させる静的型付けだからです。
- **記述量**: Python は合計を求める `sum` という部品が最初から用意され、1 行で済みます。Go は合計を自分でループして求めており、行数が増えます。書きやすさは Python、明示性は Go に出ています。
- **読みやすさの方向**: Python は「やりたいこと」を短く表し、Go は「型と手順」をきっちり示します。どちらが優れているという話ではなく、**重視する価値が違う** のです。

## 注意点

短いお題では Python の手軽さばかりが目立ちますが、これは一面にすぎません。プログラムが大きくなると、Go の型宣言や明示的な **エラー処理** が「間違いを早く見つける安全網」として効いてきます。逆に試行錯誤を素早く回したい場面では Python の短さが武器になります。**規模と目的によって有利不利が入れ替わる** という見方を持っておきましょう。

## まとめ

- 同じ「合計を返す」処理でも、Python は短く、Go は型と手順を明示する。
- Python は書きやすさ、Go は明示性と安全性を重視する設計。
- どちらが上ではなく、規模と目的で有利不利が変わる。
