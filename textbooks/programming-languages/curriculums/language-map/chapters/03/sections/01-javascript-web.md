## このセクションで学ぶこと

- JavaScript がブラウザの中で動く唯一の言語として生まれたことを理解する
- Node.js の登場でサーバ側でも同じ言語を使えるようになった経緯を押さえる
- 動的型・非同期処理という JavaScript の性格と注意点をつかむ

## ブラウザの中で動くために生まれた言語

JavaScript は、もともと Web ブラウザの中でページに動きをつけるために生まれた言語です。ボタンを押したら表示が変わる、入力内容をその場でチェックする、地図をドラッグして動かす――こうした「ページ上の振る舞い」はすべて JavaScript が担っています。

ここで重要なのは、**ブラウザの中で直接動かせるプログラミング言語は、長らく JavaScript だけだった**という点です。Python や Java でどれだけ良いコードを書いても、ブラウザはそれをそのまま実行できません。ブラウザが解釈できるのは JavaScript なので、Web ページに動きをつけたいなら JavaScript を使う、というのが事実上の前提になっています。この「ブラウザの標準言語」という立場が、JavaScript が広く使われ続けている最大の理由です。

## Node.js でサーバ側にも広がった

転機になったのが **Node.js** の登場です。Node.js は、ブラウザの外でも JavaScript を実行できるようにした仕組みです。これによって、これまでブラウザ専用だった JavaScript が、サーバ側のプログラム(データベースとやり取りする、API を提供するなど)にも使えるようになりました。

結果として、**画面側(フロントエンド)もサーバ側(バックエンド)も同じ JavaScript で書ける**という状況が生まれました。たとえば小さな Web サービスを作るとき、片方を JavaScript・もう片方を別の言語にすると、言語の切り替えで頭を使います。両方を JavaScript で統一できれば、覚える言語が 1 つで済み、コードや知識を共有しやすくなります。これが、Web 開発で JavaScript が「ブラウザとサーバの両方」で選ばれる理由です。

## 動的型と非同期処理という性格

JavaScript は**動的型**の言語で、変数の型をあらかじめ宣言しません。手早く書ける一方、型の取り違えによる不具合が実行するまで気づきにくい、という弱点があります(この弱点を補うのが次のセクションの TypeScript です)。

もう一つの特徴が**非同期処理**です。Web では「サーバからデータが返ってくるのを待つ」場面が多く、JavaScript は待っている間も他の処理を止めずに進め、結果が届いたら受け取る、という書き方を基本にしています。便利な反面、処理の順番が直感とずれやすく、初学者がつまずきやすいポイントでもあります。

## まとめ

- JavaScript はブラウザの中で動く事実上唯一の言語として、Web ページの動きを担ってきた。
- Node.js によりサーバ側でも使えるようになり、フロントとバックを同じ言語で統一できる。
- 動的型で手早く書ける反面、型の取り違えや非同期処理の順番には注意が必要。
