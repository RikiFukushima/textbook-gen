## このセクションで学ぶこと

- 代表的な用途ごとに第一候補となる言語を言える
- 早見表はあくまで「出発点」であって絶対のルールではないと理解する
- 同じ用途に複数の有力候補があることを知る

## 用途から言語を引く「早見表」という考え方

これまでの章では、言語そのものの特徴を一つずつ見てきました。「Go は速くてシンプル」「Python はライブラリが豊富」というように、言語を主語にして得意・不得意を整理してきたわけです。ところが実務では、考える順序が逆になることがほとんどです。先に「何を作りたいか」という用途が決まっていて、そこから「ではどの言語が向いているか」を引きます。この引き方をまとめたのが用途別の早見表です。

言語から特徴を覚えるのが「言語辞典」だとすれば、早見表は「用途辞典」にあたります。たとえば「Python って何が得意だっけ」と思い出すのではなく、「データ分析をしたいんだけど、何を使えばいい?」と用途から逆引きできるようにしておくと、実務でも学習でも判断が速くなります。辞書のように繰り返し引くものなので、まずは代表的な用途と第一候補をざっと頭に入れておきましょう。

## 代表的な用途と第一候補

下の表は、初級者がよく出会う用途と、その用途で実績・情報量が多い言語を並べたものです。

| 用途 | 第一候補 | 次点・併用される言語 |
| --- | --- | --- |
| Web アプリのフロント(ブラウザ側) | JavaScript / TypeScript | ― |
| Web アプリのサーバ側 | Python / Go | JavaScript(Node.js)・Java |
| データ分析・機械学習 | Python | R・SQL |
| スマホアプリ(iPhone) | Swift | ― |
| スマホアプリ(Android) | Kotlin | Java |
| 業務・大規模システム | Java | Go・C# |
| OS・組み込みなど低レイヤ | C / C++ | Rust |
| 高い安全性が必要な低レイヤ | Rust | C++ |
| データベースへの問い合わせ | SQL | ― |

たとえば「データ分析をやりたい」なら、まず Python を調べれば外しません。「iPhone アプリを作りたい」なら Swift が事実上の標準です。「ブラウザの中で動く画面を作りたい」なら、これは選択の余地がほとんどなく JavaScript / TypeScript になります。このように、用途が決まれば候補は数個、ときには 1 つまで一気に絞り込めます。

ここで「次点・併用される言語」の列にも目を向けてください。多くの用途では第一候補が 1 つに決まりきらず、複数の有力候補が横並びになっています。Web サーバのように Python・Go・JavaScript・Java のどれもが現役で使われている領域では、どれを選んでも大きく外すことはありません。「正解が 1 つに定まらない用途がある」こと自体を知っておくのも、この表の大切な読み方です。

## 早見表はあくまで出発点

注意したいのは、この表は「これ以外は使ってはいけない」という意味ではないことです。表に載っている第一候補は、初級者が学習材料や情報を集めやすく、求人や仲間も多い「無難な選択」を示しているにすぎません。

実際には、すでにチームが別の言語を使っている、既存のコードがある、といった事情で候補が変わります。同じ Web サーバでも Python・Go・Java・JavaScript のどれも現役で使われています。表の役割は「ゼロから一気に 1 つに絞る」ことではなく、「無数の選択肢を数個の有力候補まで減らす」ことだと考えてください。最終的にどれを選ぶかは、次のセクションの判断手順で詰めていきます。

## まとめ

- 実務では「用途 → 言語」の順で引くのが基本で、その地図が早見表です。
- 用途が決まれば候補は数個まで絞れます(例: データ分析なら Python)。
- 表は絶対のルールではなく、有力候補を減らすための出発点です。
