## このセクションで学ぶこと

- このカリキュラムで身についた「言語を見る目」を振り返る
- 次に学ぶ Go / Python の基礎カリキュラムへの道筋を描ける
- 学んだあとに「言語の地図」をどう使い続けるか理解する

## ここまでで手に入れた「言語を見る目」

このカリキュラムでは、個々の言語を深く触る前に、言語全体を俯瞰する地図を描いてきました。第 1 章で速度・型・エコシステム・学習コストという比較軸を持ち、第 2 章から第 4 章で代表的な言語の得意・不得意を見て、この章で用途から言語を引く方法と選び方の手順を整理しました。

これで、知らない言語に出会っても「どの軸で見ればいいか」「どんな用途に向きそうか」を自分で推測できるようになっているはずです。たとえば求人で見慣れない言語名を見かけても、「コンパイル型か」「型はあるか」「どんな用途で使われているのか」と問いを立てれば、おおよその位置づけが掴めます。これは一つの言語を覚えること以上に、長く使える力です。技術は移り変わりますが、比較軸そのものはなかなか古びないからです。

## 次に学ぶもの ― まずは手を動かす

地図が描けたら、次は実際に 1 つの言語で手を動かす番です。本シリーズでは、初学者が始めやすい Go と Python の基礎カリキュラムを用意しています。

- **Python の基礎**: 文法がやさしく、データ分析や自動化に直結します。目的が定まっていない人の最初の 1 言語にも向きます。
- **Go の基礎**: シンプルな文法で、速くて堅牢なサーバを書けます。「速さとシンプルさ」を体感したい人におすすめです。

前のセクションの考え方どおり、まずはどちらか 1 つに絞って取り組みましょう。

## 地図はこれからも使い続ける

最後に一つ。今回描いた地図は、学習が進んでも捨てずに更新し続けるものです。新しい言語やフレームワークに出会うたびに「この軸で見るとどうか」「どの用途に置けるか」と地図に書き足していけば、技術選定の判断はどんどん速く・確かになっていきます。

## まとめ

- 比較軸と用途別の見方を身につけ、未知の言語も推測できる力がつきました。
- 次は Go か Python の基礎で、1 つに絞って実際に手を動かします。
- 言語の地図は学習後も更新し続けることで、選定の判断力が育ちます。
