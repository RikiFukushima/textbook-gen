## このセクションで学ぶこと

- var による変数宣言と型の指定方法
- := による短縮宣言の使いどころ
- Go が型を自動で推論するしくみ(型推論)

## var で変数を宣言する

Go では変数を使う前に必ず「宣言」します。もっとも基本的な書き方が `var` キーワードです。`var 変数名 型` の順で書き、必要なら初期値を続けます。

```go
var name string = "Gopher"
var age int = 5
```

Go は **静的型付け** の言語なので、変数にはあらかじめ型が決まっています。一度 `string` として宣言した変数に、後から数値を入れることはできません。この性質のおかげで、型の取り違えによるバグをコンパイル時に発見できます。

初期値を書かずに宣言することもできます。その場合、変数には型ごとに決まった **ゼロ値** が入ります。

```go
var count int       // 0 が入る
var message string  // "" (空文字) が入る
var done bool       // false が入る
```

Go には「未初期化で中身が不定」という状態がありません。宣言した変数は必ず意味のある初期値を持つため、初期化忘れによる不具合が起きにくいのが特徴です。

## 型推論で型を省く

初期値を与えるなら、型の記述は省略できます。コンパイラが右辺の値を見て型を決めてくれるからです。これを **型推論** と呼びます。

```go
var name = "Gopher" // string と推論される
var age = 5         // int と推論される
```

`"Gopher"` は文字列なので `name` は `string`、`5` は整数なので `age` は `int` になります。型を書かなくても、変数の型はきちんと固定される点が重要です。動的型付け言語のように「途中で型が変わる」ことはありません。

## := はもっと手軽な短縮宣言

関数の中では、`var` すら省いた **短縮変数宣言** が使えます。`:=` を使うと、宣言と初期化を一度に書けます。

```go
func main() {
    name := "Gopher" // var name = "Gopher" と同じ意味
    age := 5
    fmt.Println(name, age)
}
```

実務の Go コードでは、関数内のローカル変数はほとんど `:=` で書かれます。短く読みやすいためです。一方 `var` は、初期値を持たせずゼロ値から始めたいときや、後述するパッケージレベル(関数の外)での宣言に使われます。

## 注意点

`:=` には 2 つの落とし穴があります。1 つ目は **関数の外では使えない** こと。パッケージレベルの変数は必ず `var` で宣言します。2 つ目は、Go では **宣言したのに使わない変数はコンパイルエラー** になることです。

```go
func main() {
    x := 10 // x を一度も使わないとエラー: declared and not used
}
```

これは「使わない変数=書き間違いやコードの残骸」とみなす Go の方針によるものです。最初は厳しく感じますが、不要な変数が混ざらず、コードが常に整理された状態に保たれます。

## まとめ

- `var` は型を明示でき、関数の外でも使える基本の宣言方法。初期値なしならゼロ値が入る。
- 初期値があれば型推論で型を省略でき、関数内では `:=` でさらに短く書ける。
- 宣言して使わない変数はコンパイルエラーになる。
