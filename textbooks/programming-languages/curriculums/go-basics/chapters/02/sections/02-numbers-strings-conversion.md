## このセクションで学ぶこと

- 数値・文字列・真偽値という基本型の種類
- Go では暗黙の型変換が行われないこと
- 明示的な型変換と文字列・数値の相互変換の方法

## 基本型をおさえる

Go でよく使う基本型は、大きく分けて 3 種類です。

- **数値**: 整数の `int`、小数の `float64` が代表格です。整数にはビット幅を固定した `int32` `int64`、符号なしの `uint` などもあります。
- **文字列**: `string`。ダブルクォートで囲んで書きます。
- **真偽値**: `bool`。`true` か `false` のどちらかを取ります。

```go
var price int = 1200
var rate float64 = 0.08
var title string = "Go入門"
var available bool = true
```

`int` の幅は実行環境によって 32bit か 64bit になります。普段は `int` で十分ですが、扱う値の範囲を厳密に決めたいときは `int64` のようにサイズを明示します。小数を扱うときは、特別な理由がなければ `float64` を選んでおけば間違いありません。

## Go は暗黙の型変換をしない

ここが Go の大きな特徴です。多くの言語では `int` と `float64` を混ぜて計算できますが、**Go では型が違うとそのままでは計算できません**。

```go
var a int = 3
var b float64 = 1.5
// fmt.Println(a + b) // コンパイルエラー: 型が違う
```

異なる型をまたいで計算するには、**明示的型変換** で型をそろえる必要があります。`型名(値)` の形で書きます。

```go
var a int = 3
var b float64 = 1.5
result := float64(a) + b // a を float64 に変換してから足す
fmt.Println(result)      // 4.5
```

一見面倒ですが、これは「意図しない型の混在によるバグ」を防ぐための設計です。たとえば整数同士の割り算が勝手に小数になる、といった分かりにくい挙動が起きません。型を変換する箇所がコード上に明示されるため、計算の意図が読み手に伝わります。

## 文字列と数値の変換は strconv で

数値を文字列にしたり、その逆をしたりする場面は実務で頻繁にあります。注意したいのは、`int` を `string()` で変換しても **数字の文字列にはならない** ことです(文字コードとして解釈されてしまいます)。文字列と数値の相互変換には標準ライブラリの **strconv** を使います。

```go
import "strconv"

n, err := strconv.Atoi("42") // 文字列 "42" → 数値 42
s := strconv.Itoa(42)        // 数値 42 → 文字列 "42"
```

`Atoi`(ASCII to integer)は変換に失敗する可能性があるため、結果とあわせて `err` を返します。たとえば `"abc"` を渡すと変換できず、`err` にエラーが入ります。ユーザー入力など「数値とは限らない文字列」を扱うときは、この `err` を必ず確認しましょう。

## 注意点

数値リテラルの扱いにも気をつけます。`10 / 3` のように整数同士で割ると結果も整数になり、`3` が返ります(小数点以下は切り捨て)。小数の結果がほしいなら、`10.0 / 3.0` のように小数として書くか、`float64()` で変換してから割ります。型を意識せずに書くと、想定と違う計算結果になりがちな箇所です。

## まとめ

- 基本型は数値(`int` / `float64`)・文字列(`string`)・真偽値(`bool`)。
- Go は暗黙の型変換をしないため、型が違う値は `型名(値)` で明示的に変換する。
- 文字列と数値の相互変換は `strconv` を使い、変換失敗の `err` を確認する。
