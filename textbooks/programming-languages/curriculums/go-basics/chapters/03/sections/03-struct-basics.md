## このセクションで学ぶこと

- `struct` で関連する複数のデータを 1 つの型にまとめられる
- 構造体の値を初期化し、フィールドにアクセスできる
- フィールド名の大文字小文字が公開・非公開を決めることを理解する

## 関連するデータを束ねる

プログラムでは「名前と年齢とメールアドレス」のように、いくつかのデータをひとまとめで扱いたい場面が頻繁にあります。Go ではこうしたデータの集まりを **構造体(struct)** で表現します。`type` と `struct` を組み合わせて、新しい型として定義します。

```go
type User struct {
    Name  string
    Age   int
    Email string
}
```

`User` という型は、`Name`・`Age`・`Email` という 3 つの **フィールド** を持ちます。これで「ユーザー 1 人分」をひとつの値として扱えるようになり、関数の引数や戻り値としてもまとめて受け渡せます。

## 値を作ってフィールドを使う

構造体の値は、フィールド名を指定して初期化するのが読みやすく安全です。作った値のフィールドには、ドット `.` でアクセスします。

```go
u := User{
    Name:  "山田",
    Age:   28,
    Email: "yamada@example.com",
}

fmt.Println(u.Name) // 山田
u.Age = 29          // 書き換えもできる
```

フィールド名を省いて `User{"山田", 28, "yamada@example.com"}` と順番だけで書くこともできますが、フィールドが増えたときに壊れやすいため、**フィールド名付きの初期化** を推奨します。指定しなかったフィールドは型ごとの初期値(`int` なら 0、`string` なら空文字)になります。

ポインタが欲しいときは `&User{...}` と書くと、構造体への参照を得られます。大きな構造体を関数に渡すときや、関数の中で値を書き換えたいときに使います。

## 公開と非公開を決める大文字小文字

Go では識別子の先頭が **大文字なら公開(エクスポート)**、**小文字なら非公開** という共通ルールがあります。これは構造体のフィールドにもそのまま当てはまります。

```go
type Account struct {
    ID       int    // 公開: 他パッケージから読める
    password string // 非公開: 同じパッケージ内だけ
}
```

`ID` はパッケージの外からアクセスできますが、`password` は同じパッケージの中からしか触れません。隠したい内部データは小文字で始めることで、不用意な書き換えを防げます。

## 注意点

- フィールド名のない位置指定の初期化は順序に依存するため、フィールドを追加・並べ替えると黙って壊れます。基本はフィールド名付きで書きましょう。
- 構造体は値としてコピーされます。関数に渡すと中身が複製されるため、元を書き換えたいときはポインタ `*User` を使います。
- 大文字小文字の公開ルールは「アクセス修飾子の代わり」です。`public` / `private` のようなキーワードは Go にはありません。

## まとめ

- `struct` は関連するフィールドをまとめて 1 つの型にする仕組み。
- フィールド名付きで初期化し、ドット記法でアクセスするのが基本。
- 先頭が大文字なら公開、小文字なら非公開という可視性ルールが効く。
