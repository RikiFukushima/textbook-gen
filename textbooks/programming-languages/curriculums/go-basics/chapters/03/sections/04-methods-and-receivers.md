## このセクションで学ぶこと

- 構造体に紐づく関数(メソッド)を定義できる
- 値レシーバとポインタレシーバの違いを理解して使い分けられる
- データと振る舞いをまとめる Go らしい設計を体験する

## メソッドとは ― 型に振る舞いを持たせる

前のセクションで作った構造体は「データのまとまり」でした。そのデータに対する操作、つまり **振る舞い** を結びつけたいときに使うのが **メソッド** です。メソッドは関数の一種ですが、`func` と関数名の間に **レシーバ** と呼ばれる特別な引数を書く点が普通の関数と違います。

```go
type User struct {
    Name string
    Age  int
}

// (u User) がレシーバ。User 型に Greet メソッドを紐づける
func (u User) Greet() string {
    return "こんにちは、" + u.Name + "です"
}
```

レシーバ `(u User)` は「このメソッドは `User` 型の値 `u` に対して動く」という意味です。メソッドの中では、`u.Name` のようにレシーバ越しにフィールドへアクセスできます。呼び出すときは `u.Greet()` のように、値にドットをつなげて書きます。データ(構造体)と振る舞い(メソッド)を同じ型のまわりにまとめられるのが、Go の素直な設計スタイルです。

## 値レシーバとポインタレシーバ

レシーバには 2 種類あります。`(u User)` のように書く **値レシーバ** と、`(u *User)` のように書く **ポインタレシーバ** です。この違いは「メソッドの中での変更が呼び出し元に反映されるか」を左右する、とても重要なポイントです。

値レシーバはレシーバを **コピー** して受け取るため、メソッド内でフィールドを書き換えても元の値は変わりません。一方ポインタレシーバは元の値そのものを指すため、書き換えが呼び出し元に反映されます。

```go
// 値レシーバ: 元の Age は変わらない
func (u User) AddAgeBad() {
    u.Age++ // コピーを変更しているだけ
}

// ポインタレシーバ: 元の Age が変わる
func (u *User) AddAge() {
    u.Age++
}

u := User{Name: "山田", Age: 28}
u.AddAge()         // u.Age は 29 になる
fmt.Println(u.Age) // 29
```

`u.AddAge()` のように、変数が値であってもポインタレシーバのメソッドは Go が自動でアドレスを取って呼んでくれます。状態を変更するメソッドや、大きな構造体を扱うメソッドでは、コピーを避けるためにポインタレシーバを選ぶのが基本です。

## 注意点

- 1 つの型のメソッドでは、値レシーバとポインタレシーバを **混在させない** のが慣習です。どこかでポインタレシーバを使うなら、その型のメソッドはすべてポインタレシーバに揃えると一貫性が保てます。
- メソッドを定義できるのは、自分のパッケージで定義した型だけです。`int` や他パッケージの型に直接メソッドは追加できません(`type` で自前の型を作れば可能です)。
- 値を変更しない小さなメソッドは値レシーバでも構いませんが、迷ったらポインタレシーバにしておくと安全です。

## まとめ

- メソッドは `func (レシーバ 型) 名前() { ... }` の形で型に振る舞いを紐づける。
- 値レシーバはコピー、ポインタレシーバは元の値を指し、変更を反映できる。
- 状態を変えるメソッドや大きな構造体ではポインタレシーバを使うのが基本。
