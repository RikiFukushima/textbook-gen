## このセクションで学ぶこと

- `func` キーワードで関数を定義し、引数と戻り値の型を書ける
- 複数の値をまとめて返す Go ならではの書き方を理解する
- 名前付き戻り値や可変長引数といった便利な構文を使える

## 関数の基本形

Go の関数は `func` キーワードで定義します。関数名のあとに括弧で引数を並べ、その後ろに戻り値の型を書くのが基本の形です。引数も戻り値も「名前のあとに型」という順序で書く点が、Go の宣言の一貫したスタイルです。

```go
func add(a int, b int) int {
    return a + b
}
```

`a int, b int` のように同じ型が続く場合は、`func add(a, b int) int` とまとめて書くこともできます。戻り値がない関数は、戻り値の型を省略します。

## 複数の値を返す

Go の大きな特徴のひとつが、関数が **複数の戻り値** を返せることです。戻り値の型を括弧で囲んで並べるだけで、2 つ以上の値を同時に返せます。

```go
func divide(a, b int) (int, int) {
    quotient := a / b
    remainder := a % b
    return quotient, remainder
}

q, r := divide(17, 5) // q = 3, r = 2
```

この仕組みは Go のいたるところで使われます。とくに「計算結果」と「エラー」をペアで返すのが Go の定番です。次のセクションで扱う `error` 型の処理は、この複数戻り値が土台になっています。受け取る側は、複数の変数を `:=` でまとめて宣言できます。

不要な戻り値があるときは、ブランク識別子 `_` で受け捨てられます。たとえば商だけ欲しいときは `q, _ := divide(17, 5)` と書きます。

## 名前付き戻り値と可変長引数

戻り値にあらかじめ名前を付けておく **名前付き戻り値** も使えます。関数の入り口でその名前の変数が初期値で用意され、`return` を引数なしで書くと現在の値がそのまま返ります。

```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return // x, y を返す
}
```

また、引数の個数を固定したくない場合は **可変長引数** を使います。型の前に `...` を付けると、その引数は呼び出し側で 0 個以上いくつでも渡せます。関数の中ではスライスとして扱われます。

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum(1, 2, 3) // 6
```

## 注意点

- 名前付き戻り値は短い関数では読みやすくしますが、関数が長くなると「どこで値が変わったか」が追いにくくなります。多用は避け、短い関数にとどめるのが無難です。
- `return` で返す値の個数と型は、宣言した戻り値とぴったり一致させる必要があります。足りない・多いとコンパイルエラーになります。
- 可変長引数は関数の引数リストの最後にしか置けません。

## まとめ

- 関数は `func 名前(引数 型) 戻り値型 { ... }` の形で定義する。
- 戻り値を括弧で並べると複数の値を同時に返せ、結果とエラーのペアが定番。
- 名前付き戻り値・可変長引数・ブランク識別子 `_` を場面に応じて使い分ける。
