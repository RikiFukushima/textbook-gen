## このセクションで学ぶこと

- Go を自分の環境にインストールする手順を理解する
- go version でインストールを確認できる
- go コマンドの主要なサブコマンドの役割を把握する

## Go をインストールする

Go を使うには、まず公式サイト([go.dev/dl](https://go.dev/dl/))から、自分の OS(Windows / macOS / Linux)に合ったインストーラーを入手します。基本的にはダウンロードしたインストーラーを実行し、案内に従って進めるだけで完了します。

- **Windows / macOS**: 公式の `.msi` / `.pkg` インストーラーを実行します。多くの場合、Go が自動で `PATH` に登録されます。
- **macOS / Linux**: Homebrew(`brew install go`)など、各 OS のパッケージマネージャーを使う方法も手軽です。

インストールが終わったら、ターミナル(コマンドプロンプト)を **開き直して** から確認します。`PATH` の変更は、新しく開いたターミナルにしか反映されないことがあるためです。

## インストールを確認する

ターミナルで次のコマンドを実行します。

```bash
go version
```

`go version go1.22.0 darwin/arm64` のように、バージョン番号と OS・CPU 種別が表示されれば成功です。`command not found` のように出る場合は、`go` が `PATH` に登録されていない可能性が高いので、ターミナルを開き直すか、インストール手順を見直します。

## go コマンドの役割

Go の操作は、ほとんど `go` という 1 つのコマンドに集約されています。後ろに付ける **サブコマンド** で動作を切り替える形です。よく使うものを挙げます。

| サブコマンド | 役割 |
| --- | --- |
| `go run` | ソースを一時的にビルドして即実行する |
| `go build` | 実行ファイルを生成する |
| `go mod` | モジュール(依存パッケージ)を管理する |
| `go fmt` | コードの書式を自動で整える |
| `go test` | テストを実行する |

`go run` と `go build` は次のセクション以降で実際に使います。これらをすべて覚える必要はなく、まずは「Go の操作は go コマンド 1 つで完結する」と理解しておけば十分です。`go help` と打てば、利用できるサブコマンドの一覧をいつでも確認できます。

なお、以前の Go は `GOPATH` という決まったディレクトリの中でしか開発しにくかったのですが、現在はモジュール方式が標準のため、好きな場所にプロジェクトを置けます。`GOPATH` という言葉を古い記事で見かけても、まずは気にしなくて大丈夫です。この教材でも、特別な作業フォルダを用意せず、好きな場所にファイルを置いて進めます。エディタは何でも構いませんが、Go の補完や整形が効く VS Code(拡張機能「Go」を導入)を使うと、書き間違いに気づきやすく学習がはかどります。

## まとめ

- 公式インストーラーかパッケージマネージャーで Go を導入します。
- `go version` でバージョンが表示されればインストールは成功です。
- 操作は `go` コマンドのサブコマンド(run / build / mod など)に集約されています。
