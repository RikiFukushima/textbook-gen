## このセクションで学ぶこと

- Go がどんな思想で生まれた言語かを理解する
- Go の代表的な特徴(コンパイル・静的型・並行処理)を説明できる
- Go が得意な領域と、向いていない領域を見分けられる

## Go とはどんな言語か

Go(ゴー)は、2009 年に Google が公開したプログラミング言語です。「Golang」と表記されることもあります。大規模なソフトウェアを多人数で開発する現場で、ビルドが遅い・コードが読みにくい・並行処理が書きにくいといった当時の悩みを解決するために設計されました。そのため Go は、機能を盛り込むよりも **シンプルさと実用性** を優先しています。文法の要素が少なく、誰が書いてもおおむね同じ形のコードになりやすいのが特徴です。

設計の中心にあるのは「読みやすさ」と「速さ」です。ここでいう速さは 2 種類あります。1 つはプログラムの実行速度、もう 1 つはビルド(コンパイル)にかかる時間の短さです。Go はこの両方を重視しています。

## Go の主な特徴

Go を語るうえで欠かせない特徴は次の 3 つです。

第 1 に **コンパイル言語** である点です。Go のソースコードは、実行前に機械語の実行ファイルへ変換されます。Python のように 1 行ずつ解釈しながら動かすのではないため、実行が速く、できあがった実行ファイル単体を配布するだけで動かせます。

第 2 に **静的型付け** である点です。変数や関数が扱う型がコンパイル時に決まるため、型の取り違えのような誤りを実行前に見つけられます。一方で、後述する型推論のおかげで、型をいちいち書き並べる負担は小さく抑えられています。

第 3 に **並行処理が言語に組み込まれている** 点です。`goroutine` という軽量な仕組みを使うと、複数の処理を同時並行で進めることが簡単に書けます。Web サーバーのように多数のリクエストを同時にさばく用途と特に相性が良いです。

## どんな場面で使われるか

具体的には、Go は次のような領域で広く使われています。

- **Web サーバー・API バックエンド**: 高速で同時接続に強く、サーバー用途の定番です。
- **CLI ツール**: 1 つの実行ファイルにまとまるため、配布・実行が手軽です。
- **クラウド基盤・インフラ系ツール**: Docker や Kubernetes など、有名な基盤ソフトウェアが Go で書かれています。

逆に、ブラウザ上で動かす画面まわり(フロントエンド)や、機械学習のように既存ライブラリが他言語(Python など)に集中している分野では、第一候補になりにくいのが実情です。「言語に得意・不得意がある」と知っておくと、道具選びを誤らずに済みます。

## まとめ

- Go はシンプルさと速さを重視して設計された、Google 発のコンパイル言語です。
- コンパイル・静的型付け・並行処理(goroutine)が三本柱の特徴です。
- Web サーバーや CLI ツール、クラウド基盤の領域で特に力を発揮します。
