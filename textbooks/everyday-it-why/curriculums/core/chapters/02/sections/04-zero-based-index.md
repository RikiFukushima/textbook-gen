## このセクションで学ぶこと

- プログラミングの世界が「0 番」から数え始める理由
- ダイクストラが勧めた「0 始まり + 半開区間」という数え方の作法
- 杭と間隔が 1 つずれる「フェンスポスト問題」とオフバイワンエラー

## 「最初の要素」は 1 番ではない

エクセルの行は 1 から始まり、本のページも 1 から、ビルも(日本では)1 階から。ところがプログラミングを習い始めた人が最初につまずくのが、「リストの最初の要素は 0 番」という奇妙なルールです。3 個入りのリストから「3 番」を取り出そうとするとエラーになる。なぜコンピュータの世界は、わざわざ人間の直感に逆らって 0 から数えるのでしょうか。

## 0 は「先頭からのずれ」

由来はメモリの住所計算にあります。C 言語のような低レイヤの世界では、配列はメモリ上に要素がすき間なく並んだものです。各要素の住所は「先頭の住所 + 何個分ずれているか × 1 個のサイズ」で求まります。すると先頭の要素は「先頭から **0 個分**ずれた場所」にあるので、0 と呼ぶのがいちばん計算が素直です。つまり配列の添字は「何番目」という順位ではなく、「先頭からいくつずれているか」という**ずれの量(オフセット)**なのです。番号というより、ものさしの目盛りに近い感覚です。この方式を**ゼロ始まりインデックス**(zero-based indexing)と呼びます。

これだけなら「機械の都合」ですが、人間側にも援軍がいます。1982 年、計算機科学者のエドガー・ダイクストラは「なぜ番号付けは 0 から始めるべきか」という有名な短いメモを書きました。主張の核は、範囲を「0 以上 N 未満」という**半開区間**で書くと良いことずくめだ、という点です。区間の長さがそのまま N になる、隣の区間とつなげても境界が重ならない、空っぽの区間も自然に書ける——0 から数えれば「N 個の要素は 0 以上 N 未満」ときれいに収まるのです。

```python
items = ["りんご", "みかん", "ぶどう"]  # 3 個のリスト
print(items[0])  # りんご(先頭は 0 番)
print(items[2])  # ぶどう(最後は「個数 - 1」= 2 番)
print(items[3])  # IndexError! 3 個あっても「3 番」は存在しない
```

この最後の行こそ、プログラマが何十年も戦い続けている宿敵、**オフバイワンエラー**(off-by-one error、「1 つずれ」のバグ)の典型です。

## 杭は何本? — フェンスポスト問題

オフバイワンエラーの原型は、コンピュータが生まれる前からある数え間違い、**フェンスポスト問題**(杭の問題)です。「長さ 4m の柵を、1m 間隔で杭を立てて作る。杭は何本必要?」——うっかり「4 本」と答えたくなりますが、正解は 5 本。間隔が 4 つあるなら、その両端を支える杭は 5 本要るからです。

```mermaid
graph LR
    P0["杭①"] ---|"間隔1"| P1["杭②"]
    P1 ---|"間隔2"| P2["杭③"]
    P2 ---|"間隔3"| P3["杭④"]
    P3 ---|"間隔4"| P4["杭⑤"]
```

「区間の数」と「区切りの数」は必ず 1 ずれる——このずれは境界のあるあらゆる場所に潜んでいます。「1 月 1 日から 1 月 10 日まで」は引き算なら 9 ですが、両端を含むので 10 日間。この章の最初のセクションで見た「256 通りなのに最大値は 255」もまったく同じ構図で、0 を含めて数えるぶん、個数と最大値が 1 ずれるのでした。

## 注意点 — 世界は 0 始まりと 1 始まりの混在地帯

やっかいなのは、世界が 0 始まりに統一されていないことです。エクセルの行も、ページ番号も 1 始まりですし、プログラミング言語にも Lua や R のように 1 から数える派がいます。ヨーロッパのビルは日本の 1 階を「0 階(ground floor)」と呼ぶので、エレベーターでも 1 つずれます。さらに西暦には「0 年」が存在しないため、21 世紀の始まりは 2000 年ではなく 2001 年——これも由緒正しいオフバイワンです。

エンジニアの自衛策は、ダイクストラ流に境界を「以上・未満」の半開区間で書く習慣です。それでもこのバグは絶えないので、プログラマの世界にはこんなジョークがあります。「コンピュータサイエンスの 2 大難問は、キャッシュの無効化、名前付け、そしてオフバイワンエラーである」。2 大と言いながら 3 つある——もちろん、わざと 1 つずらしてあるのです。

## まとめ

- 配列の添字は「何番目」ではなく「先頭からのずれ(オフセット)」。だから先頭は 0 番になる
- ダイクストラは「0 始まり + 以上・未満の半開区間」なら長さ・連結・空区間がきれいに書けると説いた
- 「間隔の数」と「杭の数」は 1 ずれる(フェンスポスト問題)。この 1 ずれ = オフバイワンエラーは西暦やエレベーターにも潜む
