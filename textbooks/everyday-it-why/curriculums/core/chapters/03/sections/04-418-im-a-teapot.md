## このセクションで学ぶこと

- 「私はティーポットです」という謎のエラーコード 418 の出生の秘密
- IETF が毎年 4 月 1 日にジョーク仕様書を出すという伝統
- 削除の危機から 418 を救った save 418 運動と、その後日談

## ブラウザで「ティーポット」に会いに行く

ブラウザで `google.com/teapot` にアクセスしてみてください。ティーポットの絵とともに「418. I'm a teapot.」と表示されます(スマートフォンなら端末を傾けるとお茶を注いでくれます)。**418 I'm a teapot** — 「私はティーポットです」。前のセクションまでの文法に従えば 4xx、つまりクライアントエラーの一種ですが、いったい何のエラーなのでしょうか。答えは「**ティーポットにコーヒーを淹れさせようとした**」エラーです。

## 4 月 1 日の RFC — 真面目な組織の真面目なジョーク

インターネットの技術仕様は RFC という文書で公開されますが、これを管理する IETF 周辺には、**毎年 4 月 1 日にジョーク RFC を発行する**という伝統があります。古くは 1978 年から例があり、有名なのは 1990 年の RFC 1149「鳥類キャリアによる IP データグラムの伝送」— 伝書鳩でインターネットのデータを運ぶという大真面目な仕様書です(2001 年にノルウェーの技術者グループが本当に鳩で実装し、9 パケット中 4 つの応答に成功しました)。

418 の故郷もこの伝統です。1998 年 4 月 1 日、Larry Masinter が発行した **RFC 2324**「Hyper Text Coffee Pot Control Protocol(**HTCPCP**/1.0)」は、HTTP を模してコーヒーポットを遠隔制御するプロトコルを定義しました。コーヒーを淹れる BREW メソッドなどに混じって、こう定められています — 「ティーポットにコーヒーを淹れるよう要求した場合、エラーコード 418 I'm a teapot を返すべきである」。

これは単なる悪ふざけではなく、当時の風潮への風刺でもありました。1990 年代は「何でもネットにつなぐ」文化の黎明期で、世界初の Web カメラがケンブリッジ大学のコーヒーポット監視用だった(Trojan Room coffee pot)ことは象徴的です。Masinter 自身、HTTP を本来の目的を超えて何にでも流用する流れを茶化す意図だったと語っています。ジョークは公式に続編まで生み、2014 年には紅茶向け拡張の RFC 7168(HTCPCP-TEA)が発行されました。

## 削除の危機と save 418 運動

冗談のはずの 418 は、エンジニアたちに愛され、Node.js・Go・Python など主要なライブラリにイースターエッグとして実装されていきました。ところが 2017 年 8 月、HTTP 仕様を司るワーキンググループの議長 Mark Nottingham が「418 は本物の HTTP 仕様には存在しないコードであり、番号の無駄遣いだ」として、各実装からの削除を提案します。正論です。しかしこれに対し、当時 15 歳の開発者 Shane Brunswick が抗議サイト save418.com を立ち上げ、こう訴えました — **418 は、コンピュータが今も人間によって作られていることを思い出させてくれる**。

賛同は瞬く間に広がり(**save 418 運動**)、削除提案は撤回されました。それだけではありません。騒動を受けて IETF は、418 を**今後ほかの意味に割り当てない予約番号**とすることを決め、現行の HTTP 仕様である RFC 9110(2022 年)には 418 が「使用しない(Unused)」番号として明記されました。エイプリルフールのジョークが、四半世紀を経て公式仕様に恒久の席を獲得したのです。

注意点として、418 は標準上「使われない」番号なので、業務の API がまじめな意味で返すのは避けるべきです(遊び心としてボット避けに使うサイトは実在しますが、あくまで非公式の使い方です)。それでもこの番号が教えてくれることは大きい — 仕様書は無味乾燥な規則集ではなく、設計者たちの会話と文化の産物だということです。

## まとめ

- 418 I'm a teapot は、1998 年のエイプリルフール RFC 2324(コーヒーポット制御プロトコル HTCPCP)が生んだジョークコード
- 2017 年の削除提案に対し save 418 運動が起こり、418 は「他の用途に割り当てない予約番号」として RFC 9110 に明記された
- ジョークが公式仕様に席を得た経緯は、仕様書が人間の会話と文化の産物であることを物語っている
