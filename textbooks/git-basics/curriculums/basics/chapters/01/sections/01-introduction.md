# バージョン管理とは

## このセクションで学ぶこと

- バージョン管理が解決する課題
- 手作業のファイル管理との違い

## 「最終版」が増えていく問題

ファイルを編集していると、`report_final.docx`、`report_final2.docx`、`report_最終_本当に最終.docx` のように似た名前のファイルが増えていきます。どれが最新なのか、いつ何を変えたのかが分からなくなり、間違って古いファイルを上書きしてしまうこともあります。

**バージョン管理**は、この問題を根本から解決する仕組みです。ファイルそのものを複製するのではなく、「いつ・誰が・何を・なぜ変更したか」という**履歴**を記録します。ファイルは常に 1 つで、過去の任意の時点の状態をいつでも取り出せます。

## 何がうれしいのか

具体的には、次のことができるようになります。

- 変更を加える前の状態にいつでも戻せる(操作ミスが怖くなくなる)
- いつ・なぜその変更をしたのかを後から追える
- 複数人が同じファイルを並行して編集し、後で 1 つにまとめられる

特に最後の「複数人での並行作業」は、ソフトウェア開発で Git が不可欠とされる最大の理由です。次のセクションでは、その Git がどのような仕組みでこれを実現するのかを見ていきます。

## まとめ

- バージョン管理はファイルの複製ではなく「変更履歴」を記録する仕組み
- 過去の状態への復帰・変更理由の追跡・並行作業の統合が可能になる
- Git は特に複数人の並行作業を得意とする
