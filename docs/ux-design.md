# ビューワ UX 設計

ビューワ(Next.js)の UX 方針。詳細は [`textbook-gen-spec.md`](./textbook-gen-spec.md) の 5 章を参照。Phase 2 で具体化する。

## 中核思想

**スクロール = 読む / スワイプ = 進む** で役割を完全分離。1 セクション = 1 ページ = 本のページめくりのメタファー。

## 操作モデル

- **縦スクロール**: セクション内の本文を読む。
- **横スワイプ**: セクション間を移動(1 スワイプ = 次の学習ユニット)。

## レスポンシブ

| デバイス | 表示 | 操作 |
| --- | --- | --- |
| スマホ | 1 セクション = 1 画面 | 横スワイプ |
| タブレット | 見開き 2 ページ(オプション) | 横スワイプ / タップ |
| PC | 見開き 2 ページ + サイドバー | 矢印キー / 「次へ」 |

## 主要画面(Phase 2)

- セクションビュー(本文・Mermaid・読了プログレス・位置インジケーター・次へ CTA)
- クイズビュー(1 問 1 ページ、回答後に即解説、スワイプ遷移)
- 章完了画面(達成感の演出)

## 未決事項(実装時に判断)

- Mermaid レンダリング: CSR で `mermaid.js` 動的描画を推奨(SSR/ヘッドレス回避)。
- スワイプ実装: CSS scroll-snap + framer-motion。
